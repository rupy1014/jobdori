# AI 수익화를 위한 콘텐츠 공장 자동화

## [0:00-0:15] HOOK - 완전 자동화 비전

"클릭 한 번에 콘텐츠 100개 자동 생산?

수집 → 생산 → 검증 → 개선 → 보고까지 완전 무인 공장 시스템 구축합니다."

## [0:15-0:45] PROBLEM - 부분 자동화의 한계

지난 영상들에서 배운 것:
- 3편: Playwright로 콘텐츠 자동 수집
- 4편: SubAgent로 생산 + 검증 자동화

근데 실제로 돌려보면:

**문제 1: 수작업 개입 필요**
- Playwright 실행 → 결과 확인
- 생산 Agent 실행 → 결과 확인
- 검증 Agent 실행 → 불합격 콘텐츠 수동 재작업
- 각 단계마다 내가 직접 명령어 입력

**문제 2: 반복 실행 번거로움**
- 매일 같은 작업을 수동으로 실행
- "오늘도 수집해야지, 생산해야지, 검증해야지..."
- 명령어 5개 × 매일 = 비효율

**문제 3: 품질 개선 루프 없음**
- 검증 실패하면? → 수동으로 고쳐야 함
- 개선 방향은? → 리포트 보고 내가 판단
- 자동으로 재시도하는 시스템 없음

이건 **반쪽짜리 자동화**입니다.

## [0:45-1:30] AGITATE - 진짜 자동화는 따로 있다

**일반 사람들 생각:**
"AI한테 시키면 자동화 아닌가?"

아니요.

**진짜 자동화는:**
- 내가 잘 때도 돌아가고
- 문제 생기면 스스로 해결하고
- 끝나면 결과를 나한테 알려주는 것

**지금까지 우리가 한 건:**
```
수동 → AI 보조
```

**진짜 해야 하는 건:**
```
완전 무인 공장
```

**차이가 뭐냐면:**

수동 + AI 보조:
```
1. 내가 수집 명령
2. 결과 확인
3. 내가 생산 명령
4. 결과 확인
5. 내가 검증 명령
6. 불합격 있으면 내가 수동 수정
7. 내가 다시 검증 명령
```
→ 단계마다 내가 개입

완전 무인 공장:
```
1. 아침에 명령 하나: /factory-run
2. 저녁에 문자 옴: "100개 완료, 95개 합격, 5개 개선 완료"
```
→ 내가 할 일: 명령 하나 + 결과 확인

**이게 진짜 자동화입니다.**

## [1:30-2:00] SOLUTION INTRO - 콘텐츠 공장 아키텍처

**5개 Agent를 하나의 파이프라인으로:**

```
/factory-run 명령 하나로
→ Collector Agent: Playwright 자동 수집
→ Producer Agent: Skills 기반 생산
→ Validator Agent: 품질 검증
→ Improver Agent: 불합격 자동 개선
→ Reporter Agent: 최종 리포트 + 문자 알림
```

**핵심 개념:**

1. **통합 Custom Command**: `/factory-run` 하나로 전체 실행
2. **SubAgent 오케스트레이션**: 5개 Agent 자동 순차 실행
3. **개선 루프**: 불합격 → 자동 개선 → 재검증 (최대 3회)
4. **알림 시스템**: 완료되면 문자로 결과 요약 (API는 다음 편)

**이제 실전입니다.**

### Agent 역할 분담

**1. Collector Agent (수집)**
- 역할: Playwright로 트렌드 콘텐츠 자동 수집
- 입력: 수집 대상 URL 리스트
- 출력: `raw-data/YYYY-MM-DD/` 폴더

**2. Producer Agent (생산)**
- 역할: 수집 데이터 → Skills 기반 재가공
- 입력: `raw-data/`
- 출력: `produced-content/`

**3. Validator Agent (검증)**
- 역할: 품질 검증 + 합격/불합격 분류
- 입력: `produced-content/`
- 출력: `approved-content/` + `failed-content/`

**4. Improver Agent (개선) - NEW!**
- 역할: 불합격 콘텐츠 자동 개선
- 입력: `failed-content/` + 피드백
- 출력: `improved-content/` → 재검증
- 루프: 최대 3회 재시도

**5. Reporter Agent (보고)**
- 역할: 전체 결과 요약 + 알림
- 입력: 전체 프로세스 로그
- 출력: 리포트 + 문자 알림 (개념)

```markdown
# 콘텐츠 공장 자동화 실행

당신은 Factory Master Agent입니다.
5개 SubAgent를 순차 실행하여 완전 자동 콘텐츠 생산 공장을 운영합니다.

## 실행 순서

### Phase 1: 수집 (Collector Agent)
```
/collect
```
- Playwright로 트렌드 콘텐츠 자동 수집
- 대상: 지정된 URL 리스트 (config/sources.yaml)
- 출력: raw-data/YYYY-MM-DD/
- 예상 시간: 5분

### Phase 2: 생산 (Producer Agent)
```
/produce
```
- 수집 데이터 → brand-voice.md 기반 재가공
- 입력: raw-data/최신 폴더
- 출력: produced-content/
- 예상 시간: 10분

### Phase 3: 검증 (Validator Agent)
```
/validate
```
- 품질 검증 + 합격/불합격 분류
- 입력: produced-content/
- 출력: approved-content/ + failed-content/
- 예상 시간: 3분

### Phase 4: 개선 루프 (Improver Agent)
```
while (failed-content 존재 && 재시도 < 3):
    /improve
    /validate (재검증)
```
- 불합격 콘텐츠 자동 개선
- 입력: failed-content/ + 피드백
- 출력: improved-content/ → 재검증
- 루프: 최대 3회
- 예상 시간: 5분/회

### Phase 5: 보고 (Reporter Agent)
```
/report
```
- 전체 결과 요약
- 통계: 수집/생산/합격/불합격/개선
- 리포트: claudedocs/factory-reports/YYYY-MM-DD-HH-MM.md
- 알림: 콘솔 출력 (문자 API는 다음 편)
- 예상 시간: 2분

## 실행 로그

각 Phase 완료 시 다음 정보 기록:
- 시작/종료 시간
- 처리 개수
- 성공/실패 개수
- 오류 발생 시 상세 로그

## 오류 처리

- Phase 실패 시: 3회 재시도 후 중단
- 중단 시: 현재까지 결과 리포트 생성
- 사용자에게 오류 상세 알림
```

```markdown
# Improver Agent - 콘텐츠 자동 개선

당신은 Improver Agent입니다.
검증 실패한 콘텐츠를 피드백 기반으로 자동 개선합니다.

## 입력

- `failed-content/` 폴더의 모든 파일
- 각 파일의 `_feedback.md` (검증 Agent가 생성)

## 개선 프로세스

### Step 1: 피드백 분석
```
[파일명]_feedback.md 읽기:
- 실패 이유 파악
- 구체적 수정 사항 확인
- 우선순위 정렬
```

### Step 2: 자동 수정
```
실패 항목별 자동 수정:
- 금지 표현 사용 → 대체 표현으로 자동 변경
- 구조 누락 → 해당 섹션 자동 추가
- 숫자 부족 → 구체적 숫자 자동 삽입
- 예시 부족 → 실전 예시 자동 생성
```

### Step 3: 개선 로그 생성
```
[파일명]_improved.md 생성:
- 원본 대비 변경 사항 명시
- 개선 횟수 기록 (1차/2차/3차)
- 다음 검증 대기
```

## 출력

- `improved-content/[파일명]_improved_v[N].md`
- `improved-content/[파일명]_changelog.md` (변경 로그)

## 재시도 제한

- 최대 3회 개선 시도
- 3회 실패 시: `permanently-failed/` 이동 + 수동 검토 요청

## 개선 예시

### Before (불합격)
```
"AI를 활용하면 콘텐츠 생산 효율이 향상될 수 있습니다."
```

**피드백:**
- ❌ 금지 표현: "~할 수 있습니다"
- ❌ 구체적 숫자 부족

### After (개선)
```
"AI로 콘텐츠 생산 시간을 5시간 → 30분으로 90% 단축하세요."
```

**변경 사항:**
- ✅ 금지 표현 제거
- ✅ 구체적 숫자 추가 (5시간 → 30분, 90%)
- ✅ 명령형 톤 적용 ("~하세요")
```

```yaml
reporting:
  output_dir: "claudedocs/factory-reports"
  notification:
    console: true
    sms: false  # 다음 편에서 활성화
    email: false
```
