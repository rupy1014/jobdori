# AI 수익화를 위한 콘텐츠 생산 자동화

## [0:00-0:15] HOOK - 문제 직격탄

"AI로 콘텐츠 100개 만들었는데 쓸만한 건 3개?"

품질 관리까지 자동화합니다.

## [0:15-0:45] PROBLEM - 3가지 고통

지난 영상에서 Playwright로 콘텐츠를 자동 수집했습니다.

근데 막상 수집한 데이터를 AI한테 재가공시키면:

**문제 1: 톤앤매너 제로**
- AI가 만든 콘텐츠가 "나"답지 않음
- "~할 수 있습니다" 같은 어색한 표현
- 브랜드 보이스 완전 무시

**문제 2: 브랜드 일관성 붕괴**
- 같은 프롬프트 써도 매번 결과물이 다름
- 어제 좋았던 게 오늘은 별로
- 100개 만들면 100개 다 느낌이 다름

**문제 3: 수작업 수정 지옥**
- "자동화했는데 검수에 시간 2배 걸림"
- 하나하나 읽어보고 고치고 또 고치고
- 차라리 내가 처음부터 쓰는 게 나을 것 같음

## [0:45-1:30] AGITATE - 왜 실패했나

"프롬프트 잘 쓰면 되지 않나?"

아니요.

**프롬프트만으로는 일관성 보장 불가능합니다.**

매번 긴 프롬프트 복사해서 붙여넣고,
그래도 결과물은 매번 다르고,
검수는 결국 내가 일일이 해야 하고.

"Skills 만들면 되지 않나?"

그것도 부족합니다.

**Skills도 수행 실패하거나 품질 들쭉날쭉합니다.**

왜냐면:
- Skills는 가이드라인일 뿐, 강제가 아님
- AI가 Skills를 잘못 해석할 수도 있음
- 검증 로직이 없으면 문제 있어도 모름

**핵심 문제:**

단일 AI에게 "생산+검증+보고" 다 시키면
품질 관리가 불가능합니다.

왜냐하면:
- 자기가 만든 건 자기가 검증 못함
- 객관적 판단 기준이 흐려짐
- 문제가 있어도 "이 정도면 됐다" 하고 넘어감

그럼 어떻게 해야 할까요?

## [1:30-2:00] SOLUTION INTRO - 멀티 에이전트 파이프라인

**생산 Agent, 검증 Agent, 보고 Agent 분리**

→ 멀티 에이전트 파이프라인 구축

**핵심 컨셉:**

```
수집된 데이터
→ 생산 Agent: Skills 기반 재가공
→ 검증 Agent: 가이드라인 준수 체크
→ 보고 Agent: 결과 요약 + 개선 방향 제시
→ 사람: 20%만 최종 확인
```

**3개 Agent가 각자 역할 분담:**
- 생산 Agent: 창의성 담당
- 검증 Agent: 품질 관리 담당
- 보고 Agent: 의사결정 지원 담당

이제 실전으로 들어갑니다.

## [2:00-3:30] VALUE DELIVERY 1 - Skills 작성 (톤앤매너 가이드라인)

### Step 1: Skills가 왜 필요한가?

**톤앤매너 가이드라인을 명문화하기 위해서입니다.**

단순히 "좋은 콘텐츠 만들어줘" 하면:
- AI가 매번 다르게 해석함
- 일관성 없음
- 검증 기준도 애매함

**Skills를 만들면:**
- 명확한 기준 제시
- 재사용 가능
- Agent들이 참고할 공통 가이드

### Step 2: Skills 파일 만들기
