## [0:30-1:00] PROBLEM - 콘텐츠 수집의 고통

AI 인플루언서 100명을 운영한다고 했을 때:

**가장 먼저 부딪히는 문제:**

"매일 발행할 콘텐츠는 어디서 가져오지?"

**선택지:**

1. **직접 쓴다?**
   - 100명 × 하루 3개 = 300개 콘텐츠
   - 불가능

2. **ChatGPT한테 생성만 시킨다?**
   - 근거 없는 내용 생성
   - 신뢰도 떨어짐
   - 팩트체크 필요

3. **뉴스/블로그에서 수집한다?**
   - 하나하나 복사 붙여넣기
   - 정리하는 데만 몇 시간
   - 출처 기록, 날짜 관리... 헬

**이게 왜 문제냐면:**

콘텐츠 생성 자동화만 해도 소용없어요.
**소스가 없으면** 생성할 게 없거든요.

그럼 어떻게 할까요?

## [1:00-2:00] AGITATE - 수작업 수집의 한계

**현실적인 수집 프로세스 (수작업):**

```

   → 노션이나 구글 문서에 저장

4. 카테고리 분류 (10분)
   → 이건 '테크', 이건 '라이프스타일'
```

- ❌ 매일 반복해야 함
- ❌ 정리 형식이 일관되지 않음
- ❌ 나중에 찾기 힘듦
- ❌ 출처 URL 잃어버림

**더 큰 문제:**

100명 인플루언서면:
- 각자 관심사가 다름 (테크, 뷰티, 게임, 재테크...)
- 카테고리별로 콘텐츠 수집 필요
- 하루 종일 수집만 해도 모자람

**이걸 자동화할 수 있다면?**

70분 → 5초

## [2:00-3:30] SOLUTION INTRO - MCP가 뭔지 쉽게 설명

**MCP = Model Context Protocol**

어려운 말 빼고 쉽게 말하면:

**"Claude한테 추가 능력을 주는 플러그인"**

**비유:**

- 기본 Claude Code = 스마트폰 기본 기능
- MCP 서버 = 앱 설치

스마트폰에 카카오톡, 네이버, 유튜브 앱 깔듯이,
Claude Code에 능력을 추가하는 겁니다.

**주요 MCP 서버:**

1. **Context7**
   - 역할: 공식 문서 자동 참조
   - 예: "React useEffect 쓰는 법" → 공식 React 문서에서 정확한 사용법 가져옴

2. **Sequential Thinking**
   - 역할: 복잡한 문제를 단계별로 분석
   - 예: "이 코드 왜 안 돌아가지?" → 체계적으로 원인 분석

3. **Playwright** ⭐ 오늘의 핵심
   - 역할: **웹 브라우저 자동 조작**
   - 예: 웹사이트 열기 → 내용 읽기 → 스크린샷 → 데이터 추출

**오늘 집중할 건 Playwright입니다.**

왜냐면:
- 웹에서 콘텐츠 자동 수집
- 사람이 브라우저로 하는 모든 걸 자동화
- 클릭, 스크롤, 텍스트 복사, 스크린샷

**이걸로 뭘 할 건데?**

1. 트렌드 뉴스 사이트 접속
2. 키워드로 검색
3. 기사 내용 추출
4. md 파일로 정리해서 저장

**전부 자동으로.**

## [3:30-6:00] VALUE DELIVERY 1 - MCP 서버 설치하기

### Step 1: MCP 설정 파일 위치 확인

**Mac/Linux:**
```bash
~/.config/claude-code/mcp.json
```

**Windows:**
```bash
C:\Users\[사용자명]\AppData\Roaming\claude-code\mcp.json
```

**파일이 없으면?** 만들면 됩니다.

### Step 2: mcp.json 파일 작성

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["-y", "@executeautomation/playwright-mcp-server"]
    },
    "context7": {
      "command": "npx",
      "args": ["-y", "@upachayaai/context7"]
    },
    "sequential-thinking": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-sequential-thinking"]
    }
  }
}
```

**이게 뭐냐면:**

- `playwright`: 웹 브라우저 자동화
- `context7`: 공식 문서 참조
- `sequential-thinking`: 복잡한 분석

**저장하고 VS Code 재시작.**

### Step 3: 설치 확인

Claude Code 채팅창에서:

```
사용 가능한 MCP 서버 목록 보여줘
```

그럼 Claude가:
```
현재 사용 가능한 MCP 서버:
- playwright
- context7
- sequential-thinking
```

이렇게 나오면 **설치 성공입니다.**

### Step 4: Playwright 테스트

```
playwright로 naver.com 열고 스크린샷 찍어줘
```

그럼:
1. 브라우저 자동 실행
2. 네이버 접속
3. 스크린샷 저장
4. 파일 경로 알려줌

**이게 바로 Playwright의 힘입니다.**

사람이 브라우저로 하는 모든 걸 자동화할 수 있어요.

## [11:30-12:00] 인식전환 - MCP의 진짜 가치

**핵심 깨달음:**

일반적인 생각:
```
"AI가 콘텐츠를 생성해준다"
```

진짜 AI 활용:
```
"AI가 전체 워크플로우를 자동화한다"
```

**차이가 뭘까요?**

**첫 번째**: AI를 "작가"로만 쓰는 것
- ChatGPT한테: "인스타 포스트 써줘"
- 한 번에 하나씩 생성
- 소스는 내가 직접 찾아야 함

**두 번째**: AI를 "시스템"으로 쓰는 것
- 수집부터 발행까지 전체 자동화
- 한 번 설정하면 계속 작동
- 사람은 검수와 전략만

**MCP가 왜 중요하냐면:**

기본 Claude는 "대화"만 할 수 있어요.
- 질문 → 답변
- 요청 → 생성

하지만 MCP를 추가하면:
- **Playwright**: 웹에서 데이터 수집
- **Context7**: 정확한 정보 참조
- **Sequential**: 복잡한 문제 분석

**"대화" → "행동"으로 바뀝니다.**

**실전 예시:**

MCP 없이:
```
나: "AI 트렌드 뉴스 요약해줘"
Claude: [할루시네이션 가능성]
```

MCP 있으면:
```
나: "AI 트렌드 뉴스 수집해줘"
Claude: [Playwright로 실제 사이트 접속]
        [최신 기사 5개 추출]
        [md 파일로 저장]
→ 팩트 기반, 출처 명확
```
