결국 이렇게 되죠.

"아... 이럴 바에 그냥 내가 하는 게 낫지 않나?"

## [3:00-5:00] Claude Code 실전 - 바로 따라하기

**3단계면 끝나요. 정말로.**

### 1단계: Node.js 설치 (3분)
코딩 기본 환경이에요. 설명란에 링크 남겨드릴게요.
다운로드 → 설치 → 다음 다음 다음 → 끝

### 2단계: Claude Code 설치 (2분)

입력하고 엔터 → 끝

### 3단계: VS Code 설치 (2분)
마이크로소프트 VS Code 다운로드 → 설치 → 끝

설치만 하면 끝? 아니에요. **바로 써봅시다.**

### 실전 예시: 업무 매뉴얼 만들기

**Step 1: 프로젝트 폴더 만들기**
바탕화면에 "업무_자동화" 폴더 하나 만드세요.

**Step 2: VS Code로 폴더 열기**
VS Code 실행 → File → Open Folder → "업무_자동화" 선택

**Step 3: 파일 3개 만들기**
왼쪽 파일 브라우저에서 "새 파일" 버튼 클릭해서:
- `01_업무목록.md` (내가 하는 업무들)
- `02_업무프로세스.md` (업무 처리 흐름)
- `03_참고자료.md` (자주 쓰는 링크, 정보)

**Step 4: 내용 간단히 작성**
01번 파일에 이렇게 써보세요:
```
# 내가 하는 업무들

- SNS 콘텐츠 기획
- 인스타그램 포스팅 (주 3회)
- 고객 문의 응대
- 월간 판매 보고서 작성
```

**Step 5: Claude Code 실행**
VS Code 상단 메뉴 → Terminal → New Terminal
터미널에 `claude` 입력 → 엔터

**이제 뭐가 다른지 보세요.**

Claude Code한테 물어봅니다:
"01번 파일 보고, 내 업무 중에서 AI로 자동화할 수 있는 거 추천해줘"

그럼 Claude가:
- ✅ 01번 파일을 직접 읽고
- ✅ 여러분의 업무 맥락을 이해하고
- ✅ 구체적인 자동화 방법을 제시해줍니다

ChatGPT 웹사이트에서는 불가능한 일이에요.

### 왜 `.md` 파일이냐고요?

**마크다운(Markdown)**은 사람과 AI의 중간 공유 형태예요.

- PDF/이미지: 사람은 보기 좋지만 AI가 이해 못함
- 순수 텍스트: AI는 읽기 쉽지만 사람이 보기 불편
- **마크다운**: 둘 다 편함 ✅

간단한 문법으로 구조화된 문서를 만들 수 있어요:
```markdown
# 제목
## 소제목
- 리스트 항목 1
- 리스트 항목 2

**굵게**
*기울임*
```

AI 작업할 때 가장 많이 추천되는 방식이에요.

### 실전 예시: 배달앱 기획한다면?

프로젝트 폴더 구조:
```
📁 새로운배달앱_기획/
  📄 README.md (프로젝트 목표)
  📄 01_문제정의.md (기존 배달앱 불편한 점)
  📄 02_차별점.md (우리만의 핵심 가치)
  📄 03_MVP기능.md (1차 출시 기능)
  📄 04_비즈니스모델.md (수익화 전략)
```

Claude Code한테:
"01번과 02번 파일 보고, 차별점이 실제로 문제를 해결하는지 분석해줘"

→ 두 파일을 다 읽고 논리적으로 분석해줍니다.

**역할 → 워크플로우 → 스텝 → 태스크**

### 실전 예시: 신사업 기획자

**역할**: 신사업 기획자

**워크플로우**: MVP 만들기

**스텝**:

2. MVP 범위 정의하기
3. 스토리텔링 개발하기
4. 랜딩페이지 만들기

**태스크** (스텝 1 예시):
- 타겟 고객 정의하기
- 핵심 문제 분석하기
- 솔루션 아이디어 브레인스토밍하기
- 경쟁사 조사하기

이렇게 쪼개면 AI한테 인수인계가 가능해요.

"신입사원 교육한다고 생각하고 써보세요."

왜냐면 그 신입사원이 바로 AI가 될 테니까요.

### 실전 AI 업무 자동화 4단계

**1단계: 태스크 나열하기**
머릿속에 있는 모든 업무를 최대한 작은 단위로 쪼개서 리스트로 만드세요.

**2단계: 태스크 그룹핑해서 스텝 정의**
비슷한 태스크들을 묶으면 그게 바로 스텝이 돼요.

**3단계: 스텝 연결해서 워크플로우 완성**
정형화되지 않은 업무를 정형화된 프로세스로 만드는 과정이에요.

**4단계: 워크플로우 문서화해서 AI 업무 인수인계 매뉴얼 만들기**
이 문서가 바로 AI 직원 교육 자료가 됩니다.

### "이것도 귀찮은데요?" 하시는 분들을 위한 꿀팁

제가 커스텀 명령어를 하나 만들어놨어요.

설명란에 GitHub 링크 남겨드릴게요. 다운로드 받으셔서:

1. 프로젝트 폴더 안에 `.claude` 폴더 만들기
2. 다운받은 파일 넣기
3. VS Code 터미널에서 `claude` 실행
4. `/task` 명령어 입력
5. "나는 이런 일을 하고 있어" 대화하면서 구체화

이 명령어는 **자동완성처럼 대화하면서 태스크를 정리해줘요.**

일일이 다 타이핑 안 하셔도 돼요.

### 예시 1: 유튜브 스크립트 작성

제 프로젝트 폴더 구조:
```
📁 유튜브_콘텐츠/
  📄 주제_리스트.md
  📄 스크립트_템플릿.md
  📄 과거_스크립트_1.md
  📄 과거_스크립트_2.md
```

Claude Code한테 물어봅니다:

**"주제_리스트.md에서 첫 번째 주제 보고, 스크립트_템플릿.md 형식으로, 과거 스크립트들 톤앤매너 참고해서 10분 유튜브 대본 써줘"**

그럼 Claude가:
1. 주제 파일 읽고
2. 템플릿 구조 파악하고
3. 과거 스크립트들 분석해서 내 스타일 학습하고
4. 그 스타일로 새 대본 작성

**Web UI에서는 절대 불가능한 일**이죠.

Web UI였다면:
- 주제 복붙
- 템플릿 복붙
- 과거 스크립트 2개 복붙
- "이 톤으로 써줘" 설명 추가
→ 토큰 한계 초과, 맥락 꼬임, 이상한 결과

### 예시 2: 고객 응대 자동화

프로젝트 폴더:
```
📁 고객응대_시스템/
  📄 FAQ.md (자주 묻는 질문)
  📄 응대_가이드라인.md (우리 브랜드 톤)
  📄 제품_정보.md

```

고객 문의 들어오면:

**"이 고객 문의 읽고, FAQ랑 정책 참고해서 응대_가이드라인 톤으로 답변 작성해줘"**

→ 모든 파일을 이해한 상태로 브랜드에 맞는 정확한 답변 생성

### 예시 3: 블로그 콘텐츠 시리즈
