## 핵심 스토리라인

### 썰 구조
```
1. 상황 제시 (0:00-0:12)
   "초안 빨리 달라그래서"
   "30분 안에 내야 하는 상황"

2. 방법 제시 (0:12-0:25)
   "음성 녹음으로 대충 씨부림"
   "Claude Code한테 때려줌"

3. 기술 포인트 (0:25-0:45)
   - HTML로 슬라이드 생성
   - 젠스파크 대신 HTML (수정 자유로움)
   - 디자인 자동 (다크모드, 그라데이션)
   - PDF 인쇄로 제출

4. 공감 + CTA (0:50-1:00)
   "넌 어때요?"
   "나이스~"
```

### 왜 이 구조가 효과적인가
1. **급박한 상황** - 공감대 형성 ("초안 빨리 달라는 거")
2. **의외의 해결책** - "음성 녹음으로 씨부림" = 의외성
3. **실용성** - 젠스파크 vs HTML 비교 = 선택의 이유
4. **재현 가능** - 시청자도 따라할 수 있음

## 기술 포인트 (대본에 녹아있음)

### 1. 음성 → 텍스트 → 구조화
```
"대충 씨부림" → Claude가 논리적으로 정리
30분 녹음 → 발표 슬라이드 구조
```

### 2. HTML 슬라이드 장점
```
"젠스파크 써도 되는데, HTML로 만들면 수정이 자유롭지롱"
- 코드로 수정 가능
- 스타일 자유도
- 버전 관리 가능
```

### 3. 자동 디자인
```
"다크모드에 그라데이션 텍스트. Claude가 알아서 해줌"
- 색상 팔레트 자동
- 반응형 적용
- 키보드 네비게이션
```

### 4. PDF 변환
```
"브라우저에서 PDF로 인쇄하면 끝"
- 추가 도구 불필요
- 그대로 제출 가능
```
