# 5편. 서비스 공개 - 1분 쇼츠 대본

> 랜딩페이지 만들고 실서버 배포 완료. 영상편집 때문에 테스트 부족 → 무료 공개 썰

## 타임라인

```
[0:00] 유튜브 댓글 자동 응답 시스템을 서비스로 만들어봤습니다.
랜딩페이지까지 만들어서 실서버에 배포했고요.

[0:05] 근데 무료로 공개합니다.
왜냐면요...

[0:08] 원래 서비스라는 게 테스트를 빡세게 해야 하거든?
근데 영상 편집하느라 개발 시간이 부족했다. 이거야.

[0:15] 솔직히 말하면
"이거 남이 쓰면 버그 터질 것 같은데?"
이 상태인데 그래서 무료인 거지. 테스터 모집하는 거지롱.

[0:22] 봐봐 회원가입하면 YouTube OAuth 연결되고,
채널 여러 개 딱딱딱 연결 가능해.

[0:28] 채널마다 말투 다르게 설정 가능.
A채널은 친근하게, B채널은 프로페셔널하게.
이러면 AI가 채널별로 다르게 답글 단다는 걸 알았지롱.

[0:38] 그리고 이게 핵심인데,
AI API 키를 각자 넣어서 쓰는 구조야.
그래서 서버비 0원 유지됨.

[0:45] 내가 돈 안 받아도 되는 이유가 이거지.
AI 비용은 각자 내니까.

[0:50] 아 근데 내가 만든 서비스를 남이 쓸 때 그 느낌이... 좀 무서운데, 넌 어때요?

[0:55] 버그 나면 말해줘요. 고쳐야 하니까.

[0:58] 나이스~
링크는 설명란에!
```

## 영상 구성 메모

### 화면 전환 포인트
- 0:00-0:07: 랜딩 페이지 스크롤 (로고 + 회원가입)
- 0:07-0:15: 셀카 (솔직한 표정) + 영상 편집 화면 자료
- 0:15-0:25: 코드 화면 or 버그 이미지 (유머러스하게)
- 0:25-0:32: 회원가입 + OAuth 연결 화면
- 0:32-0:38: 채널별 설정 화면
- 0:38-0:52: 설정 페이지 (API Key 입력)
- 0:52-1:00: 셀카 (두근두근 표정)

### 필요한 소스
- [ ] 랜딩 페이지 (실제 배포된 URL)
- [ ] 회원가입/로그인 화면
- [ ] 채널 목록 UI
- [ ] 채널별 설정 화면
- [ ] API Key 설정 화면
- [ ] 영상 편집 중인 화면 캡처 (자료용)
- [ ] 셀카 촬영 (솔직한 표정)

## 핵심 스토리라인

### 썰 구조
```
1. 성과 자랑 (0:00-0:07)
   "랜딩페이지까지 만들어서 실서버 배포했어요"

2. 반전 (0:07-0:15)
   "근데 무료로 공개합니다"
   "영상 편집하느라 테스트 못 했거든요"

3. 솔직함 (0:15-0:25)
   "버그 터질 것 같은데?"
   "그래서 테스트 해주실 분들 모집"

4. 기능 소개 (0:25-0:52)
   - OAuth 연결
   - 멀티 채널
   - 채널별 설정
   - API 키 각자 입력 (서버비 0원)

5. 감정 + CTA (0:52-1:00)
   "버그 나면 말해줘요"
   "두근두근해~"
```

### 왜 이 구조가 효과적인가
1. **솔직함 = 신뢰** - 완벽하지 않다고 인정하면 오히려 믿음
2. **참여 유도** - "테스트 해줘" = 시청자를 공동 제작자로
3. **진입 장벽 낮춤** - "무료니까 일단 써봐"
4. **공감대 형성** - 1인 개발자의 현실 (시간 부족)

## 4편 → 5편 연결 포인트

| 4편 | 5편 |
|-----|-----|
| 내 채널 하나 | 멀티 채널 지원 |
| 내 API 키 하드코딩 | 유저별 API 키 |
| 나만 씀 | 회원가입 시스템 |
| 터미널로 실행 | 랜딩 페이지 공개 |
| 완성! | 완성...인데 테스트 부족 |
