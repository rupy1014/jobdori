# 잡도리 루프 실전 - 1인 AI 조직의 업무 매뉴얼 만들기

## [0:00-0:30] HOOK - 오늘의 핵심

지난 영상에서 VS Code 기초, md 파일, CLAUDE.md 개념을 배웠습니다.

오늘은 **본격적인 실전**입니다.

**잡도리 루프.**

**이게 뭐냐면:**

AI를 점점 똑똑하게 만드는 반복 학습 과정입니다.

**왜 중요하냐면:**

ChatGPT한테 뭔가 시켰는데 마음에 안 드는 경험, 다들 있으시죠?

"다시 해줘" → 또 마음에 안 듦
"이번엔 이렇게 해줘" → 또 마음에 안 듦

**매번 똑같은 실수를 반복합니다.**

**잡도리 루프는 이 문제를 해결합니다.**

## [0:30-1:30] PROBLEM - 현재 AI 사용 방식의 문제

**일반적인 AI 사용 방식:**

```
작업 시도 → 마음에 안 듦 → 다시 설명 → 또 마음에 안 듦 → 반복 → 포기
```

**왜 이럴까요?**

**문제 1: AI가 내 취향을 모름**
- 매번 긴 설명 필요
- 이전 피드백을 기억 못함
- 같은 실수 반복

**문제 2: 여러 일을 한번에 시킴**
- "캐릭터 100개 만들어줘"
- "완벽한 콘텐츠 생성 시스템 만들어줘"
→ 범위가 너무 넓어서 실수 많음

**문제 3: 결과가 안 좋으면 AI 탓**
- "AI가 형편없네"
- "Claude가 내 의도를 이해 못해"
→ 사실은 우리가 명확한 지침을 안 준 것

**이게 왜 문제냐면:**

스타트업을 생각해보세요.

**최초 5명이 회사 문화를 결정합니다.**

- 명확한 업무 분장이 없으면 → 혼란
- 매뉴얼이 없으면 → 매번 물어봐야 함
- 역할이 불분명하면 → 책임 회피

**1인 AI 조직도 똑같습니다.**

AI한테 여러 일을 한번에 맡기면 실수합니다.
명확한 지침 없이 일 시키면 결과가 엉망입니다.

## [1:30-3:00] AGITATE - 제대로 안 하면 어떻게 되는가

**실제 실패 사례를 보여드립니다.**

### 케이스 1: 매뉴얼 없이 일 시킨 경우

```
"AI 인플루언서 캐릭터 100개 만들어줘"
```

**결과:**
- 나이가 전부 30대
- 직업이 다 비슷함
- 톤이 딱딱함
- 캐릭터끼리 구별 안 감

**왜 이렇게 됐을까요?**

명확한 기준이 없으니까요.

- 나이 분포는?
- 직업 다양성은?
- 톤앤매너는?
- 금지 표현은?

**아무것도 안 알려줬습니다.**

### 케이스 2: 한 번에 너무 많은 일 시킨 경우

```
"AI 인플루언서 100명 시스템 완벽하게 만들어줘.
캐릭터 설정, 콘텐츠 전략, 자동화, 발행 스케줄까지 다."
```

**결과:**
- 범위가 너무 넓어서 뭐부터 해야 할지 모름
- 대충대충 만든 티가 남
- 어디가 문제인지 파악도 어려움

**왜 이렇게 됐을까요?**

역할 분담이 안 됐으니까요.

실제 조직이었으면:
- A: 캐릭터 담당
- B: 콘텐츠 전략 담당
- C: 자동화 담당

이렇게 나눠서 했을 텐데,

AI한테는 전부 한번에 시켰습니다.

### 케이스 3: 결과 안 좋은데 AI 탓만 한 경우

```
1차 작업: "캐릭터 10개 만들어줘" → 마음에 안 듦
2차 작업: "다시 만들어줘" → 또 마음에 안 듦
3차 작업: "이번엔 제대로 해줘" → 또 마음에 안 듦

→ "AI가 형편없네. 못 쓰겠다."
```

**왜 이렇게 됐을까요?**

**우리가 명확한 기준을 안 줬으니까요.**

"마음에 안 든다"는 피드백은:
- 뭐가 문제인지 구체적이지 않음
- AI가 학습할 수 없음
- 다음에도 똑같은 실수

**진짜 문제는:**

AI가 아니라 **우리의 지침이 불명확한 것**입니다.

## [3:00-4:00] SOLUTION INTRO - 잡도리 루프란 무엇인가

**인식을 전환해야 합니다.**

AI는 **직원**입니다.
우리는 **관리자**입니다.

**관리자의 책임:**
- 명확한 업무 지침 제공
- 역할 분담
- 결과에 대한 피드백
- 매뉴얼 정리

**결과가 마음에 들고 안 들고는 관리자 책임입니다.**

**잡도리 루프는 이걸 체계화한 방법론입니다.**

### 잡도리 루프 핵심 원칙 3가지

**원칙 1: 작게 시작하라**
- 100개 한번에 X
- 3개 프로토타입으로 시작 O

**원칙 2: 피드백을 규칙으로 만들어라**
- "마음에 안 들어" X
- "나이가 30대만 있음. 10대 20% 필요" O
→ 이걸 CLAUDE.md에 규칙으로 저장

**원칙 3: 잘 모르는 건 매뉴얼 만들지 마라**
- 추측으로 규칙 쓰기 X
- 실제로 확인하고 "이정도면 됐다" 싶을 때 문서화 O

**자, 이제 실전으로 들어갑니다.**

## [4:00-7:00] VALUE DELIVERY 1 - 잡도리 루프 6단계

**잡도리 루프는 6단계로 진행됩니다.**

### 1단계: 작게 시작하기 (3개 프로토타입)

**Claude한테 이렇게 물어보세요:**

```
@README.md @CLAUDE.md

이 프로젝트로 뭔가 3개만 만들어줘.
작게 시작할 거야.
```

→ 100개 아니고 **3개**
→ 범위를 최소화

**왜 3개냐고요?**
- 1개: 패턴 파악 안 됨
- 2개: 여전히 부족
- **3개: 문제점 보이기 시작** ✅
- 10개: 너무 많아서 피드백 힘듦

**Claude가 뭔가 3개 만들어줄 겁니다.**

### 2단계: 결과 확인 및 문제점 발견

**생성된 결과를 보면서:**

```
✅ 좋은 점은?
- (구체적으로 적어보세요)

❌ 안 좋은 점은?
- (구체적으로 적어보세요)
```

**예시:**
```
✅ 좋은 점: 설명이 구체적
✅ 좋은 점: 형식이 일관적
❌ 문제: 톤이 너무 딱딱함
❌ 문제: 예시가 하나도 없음
```

**중요한 건 구체성입니다.**

"마음에 안 들어" (X)
"톤이 딱딱해. 친근하게" (O)

### 3단계: CLAUDE.md에 좋은 부분 규칙으로 저장

**좋았던 건 유지 규칙:**

```markdown
## 작업 규칙

### 유지 (계속 이렇게)
- 설명이 구체적 (좋았어요)
- 형식이 일관적 (좋았어요)
```

**안 좋았던 건 개선 규칙:**

```markdown
### 개선 (이렇게 바꿔)
- 톤: 친근하고 캐주얼하게
- 예시: 각 항목마다 구체적 예시 1개씩 추가

### 금지 표현
- ~습니다, ~것입니다 (너무 딱딱함)
```

CLAUDE.md 저장.

### 4단계: 규칙 기반으로 2차 작업

```
@CLAUDE.md 참고해서 3개 더 만들어줘
```

**이제 Claude가 CLAUDE.md 규칙을 따릅니다.**

결과 확인:
- 톤이 친근해짐 ✅
- 예시가 포함됨 ✅

**만족도: 1차 50% → 2차 80%**

### 5단계: 또 문제 발견하면 규칙 보강

**2차 결과 확인:**

```
✅ 좋아진 점: 톤 개선됨
✅ 좋아진 점: 예시 추가됨
❌ 새 문제: 예시가 너무 짧음
```

**CLAUDE.md 규칙 추가:**

```markdown
### 개선 (이렇게 바꿔)
- 예시: 최소 2-3줄 이상의 구체적 예시
```

### 6단계: 3차 작업 - 거의 완벽

```
@CLAUDE.md 참고해서 3개 더 만들어줘
```

**결과:**
- 톤 친근 ✅
- 예시 포함 ✅
- 예시 구체적 ✅

**만족도: 1차 50% → 2차 80% → 3차 95%**

**이게 잡도리 루프입니다.**

**3번 돌리면 95%까지 올라갑니다.**

## [7:00-9:00] VALUE DELIVERY 2 - 실전 예시: AI 인플루언서 캐릭터 생성

**실제 예시로 보여드리겠습니다.**

**2차 생성:**

```

```

**결과:**
```
character-4.md: 이민준 (22세, 게임 스트리머) ✅

character-6.md: 강태민 (26세, 여행 유튜버) ✅
```

**훨씬 나아졌습니다!**

### 3회전: 콘텐츠 생성 및 톤앤매너 개선

**캐릭터로 콘텐츠 만들어보기:**

```

```

**생성 결과:**


**문제점 발견:**
```
❌ 너무 딱딱함 ("~습니다" 말투)
❌ 22세 게임 스트리머가 이렇게 말할까?
❌ 이모지가 1개밖에 없음
❌ 흥분된 느낌이 안 남
```

**CLAUDE.md에 톤앤매너 규칙 추가:**

```markdown
## 톤앤매너 규칙

### 나이별 말투
- 10대: 완전, 진짜, ㅋㅋ, 이모지 많이 (3-5개)

### 캐릭터별 에너지

- 전문직: 신뢰감, 정보 중심, 차분함

### 금지 표현
- ~습니다, ~것입니다 (너무 딱딱함)
- ~한다, ~이다 (뉴스 같음)
```

**3차 생성 (개선된 규칙 적용):**

```
@CLAUDE.md @characters/character-4.md
```

**이게 바로 잡도리 루프입니다.**

## [9:00-11:00] VALUE DELIVERY 3 - 1인 AI 조직의 역할 분담

자, 여기서 중요한 개념 하나 더.

### 스타트업 조직 vs 1인 AI 조직

**스타트업 (최초 5명):**

```
A: 캐릭터 기획 담당
B: 콘텐츠 작성 담당
C: 플랫폼 관리 담당
D: 품질 검수 담당
E: 전략 총괄
```

→ 역할이 명확함
→ 각자 맡은 일만 잘하면 됨
→ 의사소통 명확

**1인 AI 조직 (잘못된 방식):**

```
"AI 인플루언서 시스템 다 만들어줘"
```

→ 역할 불명확
→ 범위 너무 넓음
→ 결과 엉망

**1인 AI 조직 (올바른 방식):**

```
CLAUDE.md 파일들로 역할 분담:

1. character_creator.md
   - 역할: 캐릭터 기획 전문가
   - 책임: 페르소나 생성, 일관성 유지
   - 규칙: 나이 분포, 직업 다양성, 중복 검사

2. content_writer.md
   - 역할: 콘텐츠 작성자
   - 책임: 플랫폼별 콘텐츠 생성
   - 규칙: 톤앤매너, 금지 표현, 이모지 사용

3. quality_checker.md
   - 역할: 품질 검수자
   - 책임: 일관성 검사, 오류 탐지
   - 규칙: 체크리스트, 승인 기준
```

**이렇게 하면:**

- AI한테 "character_creator.md 보고 캐릭터 3개 만들어줘"
- AI한테 "content_writer.md 보고 이 캐릭터로 콘텐츠 5개 작성해줘"
- AI한테 "quality_checker.md 보고 이 콘텐츠 검수해줘"

→ 역할이 명확함
→ 실수 줄어듦
→ 관리 쉬움

### 실전 예시: 역할별 CLAUDE.md

**character_creator.md:**

```markdown
# 캐릭터 기획 전문가

당신은 AI 인플루언서 캐릭터를 만드는 전문가입니다.

## 당신의 책임
- 고유한 페르소나 생성
- 일관성 있는 캐릭터 설정
- 중복 방지

## 반드시 따라야 할 규칙

### 나이 분포
- 10대: 20%
- 20대: 50%
- 30대: 30%

### 직업 다양성
- 프로페셔널: 40%
- 크리에이터: 30%
- 학생/취미: 30%

### 중복 검사
```
