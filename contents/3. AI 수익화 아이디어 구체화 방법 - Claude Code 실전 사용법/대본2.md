# AI에게 업무 맡기기 - 업무 흐름 만드는 연습

## [0:00-1:00] HOOK

지난 영상에서는 AI를 위한 업무 매뉴얼을 만들었습니다.
이 업무 매뉴얼을 기반으로하면 AI가 조금 똑똑해지긴 하죠.

**목표는 자동화입니다.**
근데 업무 매뉴얼만으로는 부족합니다.

**왜냐면:**
- 업무 지침 = "어떻게 할지" 구체적인 방법론
- 자동화 = "1번하고 2번하고 3번해" 단계별 실행 명령
- AI는 단계별 명확한 지시가 필요

업무 매뉴얼은 프롬프트인거고, 자동화는 프롬프트 엔지니어링 인거에요. 
업무 매뉴얼은 하나의 작업에 대한 지침인거고, 자동화까지 갈려면 일련의 업무 처리의 연쇄과정, 워크플로우가 필요한겁니다.

**그래서 오늘은 단계별 명확한 지시를 할 수 있도록 나만의 업무 흐름 만들기 연습을 할겁니다.**

**업무 흐름이란:**
- 내가 하는 일을 단계로 쪼개기
- "이거 하면 다음은 저거" 정리하기
- 머릿속 방식을 글로 표현하기

**오늘 배울 것:**

1. **커스텀 커맨드** (도구)
   - 내 방식을 명령어로 저장하는 Claude Code 기능
   - 자동화의 첫 단계

2. **업무 흐름 만들기** (핵심)
   - 내 작업을 단계로 쪼개는 연습
   - AI가 이해할 수 있게 구조화
   - 완벽함이 아닌 "정리하는 능력" 기르기

**왜 이게 필요한가?**

자동화를 하려면 **개발자처럼 사고할 수 있어야 합니다.**

개발자적 사고 = 논리적 사고

개발자는 논리적으로 사고합니다.
기능을 작은 단위로 쪼개고, 여러 상황을 미리 고려하죠.

이 사고방식을 적용해서, 나만의 업무 흐름을 만들어 봅시다.

예시를 한번 들어볼게요. 내 업무가 "초기 창업가"라고 해봅시다.
새로운 아이템이 있으면 빠르게 기획하고 바이브코딩해서 MVP를 출시하는 게 내 업무입니다.

누구나 창업 아이디어가 떠오를 수 있고, 만들고싶은 서비스가 있을겁니다.
그럴땐 여러분들은 어떻게 하시나요?

저 같은 경우는 

- 주변 사람들한테 설명해보고 반응을 보거나
- 리서치를 하면서 더 구체화를 하면서

처음 무언가로부터 영감 받은 그 느낌을 
암묵지에서 명시지로 언어화를 해보는거죠.

이런 아이디어 구체화 과정을 제 나름대로 정리해봤더니,
아래와 같이 업무를 쪼갤 수 있었습니다.

```
내 작업을 쪼갤 때:

논리적 사고 연습:
- 하나의 작업을 이루는 요소를 더 작은 단계로 쪼개서 나열
  예: "A 아이디어 구체화"
      → 1) 아이디어 스케치 (막연하게라도 어떤 서비스인지)
      → 2) 필요한 기능들 도출 (뭐가 필요할까?)
      → 3) 각 기능 구체화 (이 기능은 누가 쓸까? 어떻게 쓸까?)
      <!-- → 4) 기능 통합 및 정책 정리 (중복은? 공통 규칙은?) -->
      → 5) 타겟 고객 명확해짐 (아, 이런 사람들이 쓰겠네!)
      → 6) 타겟 중심 스토리텔링 (고객의 하루, 고민, 해결 과정)
      → 7) 세일즈 랜딩 구성 (어떻게 설득할까?)
      → 8) MVP 범위 정하기 (최소한으로 검증하려면?)

→ 이게 "업무 흐름 만들기"

실제로 위 예시는 린 스타트업/디자인 씽킹 방식과 일치하는 면이 있는데요.
여러분의 업무를 이런 식으로 논리 흐름으로 만들어 보는 겁니다.

이렇게 하면:
- 머릿속 막연한 생각이 명확한 단계로 바뀝니다
- AI에게 "이 순서대로 해줘"라고 위임할 수 있습니다
- 반복되는 작업을 자동화할 수 있게 됩니다
```

이런 식으로 업무 흐름을 정리하는 연습이 오늘의 핵심입니다.

여러분도 자주 하는 작업을 하나 골라서:
1. 평소에 어떤 순서로 하는지 관찰해보고
2. 그걸 단계별로 쪼개서 적어보고
3. 쪼갠 업무를 매뉴얼로 만들어서 AI에게 시켜보세요.

이렇게 AI에게 일련의 과정을 시켜보다 보면 업무 흐름에 체계가 생기기 시작합니다.

그럼 이 체계를 더 발전시켜서 나만의 도구를 만들어볼 수 있습니다.


**Appkit 소개**

제가 만든 아이디어 구체화 도구를 소개해볼게요.

Claude Code에는 **Custom Commands(커스텀 커맨드)**라는 기능이 있습니다.
쉽게 말해서, 자주 사용하는 업무 흐름을 단축키처럼 저장해두고
슬래시(/) 명령어로 바로 실행할 수 있는 기능입니다.

Appkit은 GitHub에 공개해뒀는데요.
영상 더보기란에 링크를 남겨둘 테니 한 번 실습해보세요.
Git을 잘 모르시는 분은 **Download ZIP**으로 다운받으시면 되고,
Git을 사용하실 수 있는 분은 해당 링크를 복사해서 터미널에서 `git clone`하시면 됩니다.

함께 해볼까요?
터미널을 여신 다음 원하시는 폴더로 이동해서 `git clone [주소]`를 입력하면 프로젝트가 생성됩니다.
VS Code를 열면 `.claude/commands` 폴더에 여러 명령어 파일들이 있는 걸 볼 수 있습니다.

먼저 VS Code에서 터미널로 Claude를 띄워보겠습니다.
Claude Code가 뜨면 커스텀 명령어를 슬래시 명령어로 사용할 수 있습니다.

**`/appkit.new` - 아이디어 스케치**

`/appkit.new` 명령어로 만들고 싶은 아이디어를 적어봅니다.
이때 **"무엇"**보다는 **"왜"** 이런 걸 만들어야 하는지 관점으로 적어주세요.

예를 들어:
> "AI 인플루언서를 만들어서 자동으로 포스팅하고, DM이나 댓글에 자동 반응하고 싶어. 여러 페르소나를 설정해서 팔로워를 늘리고, 나중에 광고도 받아서 진행하고 싶어."

이렇게 적으면 결과가 나옵니다:
- **서비스 컨셉**: AI 인플루언서 네트워크
- **초기 세팅**: 100개 페르소나 생성, 프로필 이미지 제작, 톤앤매너 설정
- **콘텐츠 운영 방법**
- **광고 수익화 방법**
- **MVP 로드맵**

1번부터 10번까지 굵직한 Feature들이 나열됩니다.
수익 모델부터 MVP 방향까지 전체적인 아웃라인을 스케치해줍니다.

이건 한 번에 완성되기보다는 **대화를 통해 구체화**하는 과정입니다.
Claude한테 "이대로 진행할지", "내 생각과 맞는지" 이야기하면서 조금씩 발전시키면 됩니다.
"이런 식으로 진행해줘"라고 하면 각각의 큰 Feature별로 구체적인 스펙을 만들어줍니다.

그러면 1번부터 10번까지 상세하게 구체화된 파일이 생성됩니다.
예를 들어 1번에서 페르소나를 어떻게 만들지 상세하게 적혀 있는 걸 볼 수 있습니다.

---

**`/appkit.spec` - 기능 상세화**

만약 001 페르소나 생성 방법이 마음에 들지 않으면,
`/appkit.spec` 명령어로 해당 번호를 지정해서 더 구체화할 수 있습니다.

**병렬 작업 팁**:
한 세션에서 1번(001) 파일을 작업하는 동안,
다른 세션을 열어서 2번, 3번 스펙을 별도로 요청할 수 있습니다.
각 세션이 같은 파일을 작업하지 않으면 효율적으로 병렬 처리가 가능합니다.
Pro 사용자는 한 번에 5개 세션까지 허용되니, 병렬로 처리하시는 걸 권장드립니다.

**`/appkit.customer` - 타겟 유저 정의**

스펙이 구체화된 기능들을 중심으로 타겟 유저를 명확하게 해봅니다.
어떤 유저가 쓸 것인지, 해당 타겟 유저의 페르소나를 설정하고 스토리텔링을 하는 과정입니다.

결과물로 커스텀 페르소나와 사용자 경험(User Journey)이 나옵니다.
어떤 사용자들이 내 아이디어를 사용할지,
이 사람들을 어떻게 설득할 수 있을지 이 단계에서 구체화할 수 있습니다.

내용을 보시면 소설처럼 적혀 있죠.
스토리텔링이기 때문에 사용자가 최대한 구체적으로 참고할 수 있도록 나열되어 있습니다.

**왜 스토리텔링이 중요한가?**
해커톤을 나간다고 했을 때, 어떤 제품을 만드는지도 중요하지만
이 제품을 **어떻게 어필할 것인지**도 중요합니다.
타겟 고객에게 처음 아이디어를 설득할 수 있어야 하는데,
설득 기법 중 하나가 스토리텔링입니다.
타겟 고객이 이 아이템에 어떻게 어필될지,
청중의 마음을 어떻게 사로잡을지 이 과정에서 다듬어집니다.

**`/appkit.sales` - 세일즈 포인트 도출**

전 단계에서 스토리텔링이 완성되었다면, 여기서 셀링 포인트를 뽑을 수 있습니다.
타겟 고객들에게 우리 제품을 어떻게 판매할지 설득하는 과정입니다.
구체적으로 랜딩 페이지, 홈페이지, 리플렛, 팜플렛 형태 등이 나올 수 있습니다.

**`/appkit.mvp` - MVP 범위 정하기**

지금까지 내 창업아이템에 대해 **누가, 왜** 필요한지가 구체화되었다면,
이 아이디어를 검증하기 위해 **최소한 만들어야 되는 기능이 뭔지** 범위를 정합니다.
이것이 MVP(Minimum Viable Product)입니다.

MVP 범위와 MVP 매트릭스를 산출하게 되고,
이걸 기반으로 바이브 코딩으로 넘어갈 수 있습니다.

**실습: 랜딩 페이지 만들기**

산출된 파일을 보시면 세일즈 랜딩 페이지 기획안도 나온 걸 볼 수 있는데요.
이 기획안을 가지고 실제로 랜딩 페이지를 만들어보겠습니다.

Claude Code를 통해서 만들어도 되지만,
최근(11월 20일 기준) **Gemini 3**가 발표되었습니다.
Gemini 3가 랜딩 페이지를 기가 막히게 만들어주기 때문에 이걸로 만들어볼게요.

1. Gemini 사이트를 방문합니다
2. 아까 자동으로 만들어진 sales-landing.md 파일을 복사합니다
3. **Canvas(캔버스)**를 선택합니다
4. 우측에서 **Gemini 3 사고 모드**를 선택합니다
5. MD 파일을 붙여넣고 "구현해줘"라고 입력합니다
6. 캔버스가 열리면서 Gemini가 HTML을 코딩해주는 걸 볼 수 있습니다
7. 결과가 완성되면 **공유 버튼**을 누르면 실제 링크가 나옵니다

이 링크로 느낌을 보는 거죠.
이런 서비스가 나오면 어떨지.
직접 MVP를 제작해봐도 좋고,
마음이 맞는 개발자나 디자이너나 팀원을 꾸려서 진행해도 됩니다.

**마무리**

하지만 요즘 시대는 **아이디어만 있으면 누구나 창업할 수 있는 시대**입니다.
여러분들이 먼저 하고 있는 업무부터 자동화하고,
제 강의를 통해 나중에 바이브 코딩까지 하시면
혼자서도 여러 서비스, 여러 회사를 운영할 수 있게 되실 겁니다.

오늘의 핵심은 **나만의 업무 흐름 만들기**입니다.
업무 흐름을 만들기 위해서 머릿속에 복합적으로 있었던 업무를 최대한 언어화하면서,
암묵지에 있었던 비정형 업무 프로세스를 명시지로 꺼내는 연습을 해보셔야 합니다.

단순히 오늘은 저의 Appkit 예시를 소개드렸지만,
**Appkit 자체가 중요한 게 아니고 여러분만의 업무를 워크플로우로 만드는 것 자체가 중요한 겁니다.**
Appkit은 하나의 샘플인 거죠.

제가 만든 커스텀 명령어를 참고하셔서 여러분들만의 도구를 만드시라고 링크를 공유드릴 테니,
최대한 저의 노하우와 경험을 가져가시길 바랍니다.

감사합니다.
