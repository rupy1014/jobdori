## [0:45-2:00] PROBLEM - 패턴 발견했는데 저장 못함

### 2편 이후, 패턴이 보임

**2편에서 업무 매뉴얼 써봤죠?**

```
영상 1: "타겟 정하고 → 페인 포인트 → 후킹"
영상 2: "타겟 정하고 → 페인 포인트 → 후킹"
영상 3: "타겟 정하고 → 페인 포인트 → 후킹"

→ "어? 나 항상 똑같은 순서로 하네?"
```

**비유: 마나 감응**

```
무의식 → 감지 → 컨트롤

1단계: 내 패턴 모름
2단계: "나 항상 이렇게 하네?" (패턴 감지)
3단계: 패턴 저장하고 재사용 (의식화)
```

### 진짜 문제

**패턴은 발견했는데, 고정할 방법을 모름**

```
욕구:
- 이 패턴을 저장하고 싶음
- 단축키처럼 쓰고 싶음
- "/영상기획 [주제]" 이런 식으로
```

**필요한 것:**
1. 패턴 고정하기 → 논리 사이클로 만들기
2. 빠르게 실행하기 → 커스텀 커맨드로 만들기

### 업무지침 vs 커맨드

```
업무지침 (rules/영상기획.md):
- 영상 기획 방법 상세히 적혀있음
- 사용: "rules/영상기획.md 보고 운동 앱 기획해줘"
→ 작동함 ✅
→ 매번 긴 명령 ❌

커맨드 (.claude/commands/영상기획.md):
- 업무지침을 실행하는 명령어
- 사용: "/영상기획 운동 앱"
→ 간단 ✅
→ 빠름 ✅
```

**비유:**
업무지침 = 100페이지 매뉴얼
커맨드 = F5 단축키

## [2:00-3:00] SOLUTION INTRO - 논리 사이클이란?

### 논리 사이클 = 내 작업의 단계별 흐름

**예시: 영상 기획 사이클**
```
1. 주제 명확화
2. 타겟 정의
3. 페인 포인트 3개
4. 후킹 포인트
5. 핵심 메시지
6. 썸네일 컨셉
7. 제목 후보
```

### 왜 필요한가?

1. **AI는 명확한 지시만 이해**
   - ❌ "영상 기획 잘해줘"
   - ✅ "1. 타겟 → 2. 페인 포인트 → 3. 솔루션"

2. **암묵지를 명시지로**
   - 내가 하면: 무의식적으로
   - AI 시키려면: 명시적으로

3. **4편(자동화) 준비**
   - 사이클 없으면 위임 불가
   - 사이클 엉망이면 자동화 엉망

## [3:00-4:00] AGITATE - 완성도가 성능 결정

### 업무지침의 진화

```
1단계: 작업 참고서 (2편)
→ "이렇게 하세요" 설명

2단계: 논리 사이클 (3편, 오늘)
→ "1→2→3 순서로" 단계

3단계: 자동화 워크플로우 (4편)
→ 담당자에게 완전 위임
```

**핵심:**
> **"업무 매뉴얼 완성도 = AI가 얼마나 잘 일하는가"**

실제 사례를 봅시다. **appkit**입니다.

## [4:00-6:00] VALUE 1 - 내 업무 사이클 만들기

### 3단계 연습

**1단계: 반복 작업 찾기**
- 유튜브 영상 기획
- 블로그 작성
- 고객 제안서
- 아이디어 검증

예시: "유튜브 영상 기획"

**2단계: 방식을 단계로 쪼개기**

처음엔 막연함:
```
"영상 기획? 그냥... 적당히..."
```

관찰:
```
"나는 항상 타겟부터 생각하네"
"그 다음 페인 포인트 찾고"
"후킹 만들고..."
```

단계로 정리:
```
1. 주제 명확화
2. 타겟 시청자 정의
3. 페인 포인트 3개
4. 후킹 포인트
5. 핵심 메시지
6. 썸네일 컨셉
7. 제목 후보
```

**중요:** 정답 찾기 ❌ / 내 방식 정리하기 ✅

**3단계: 커맨드로 만들기**

파일: `.claude/commands/영상기획.md`

```markdown
## 입력
주제: $ARGUMENTS

## 내 영상 기획 사이클

### 1단계: 주제 명확화
입력을 구체적으로 변환

### 2단계: 타겟 정의
- 연령대, 직업군
- 3가지 페인 포인트

### 3단계: 페인 포인트 구체화
1. [문제 1]
2. [문제 2]
3. [문제 3]

### 4단계: 후킹 포인트
- 숫자형
- 질문형
- 반전형

### 5-7단계: 메시지, 썸네일, 제목
[각 단계별 상세]

## 출력
`contents/[주제]/기획.md` 저장
```

사용:
```
/영상기획 운동 앱
→ 1-7단계 자동 실행
```

### 핵심

**완벽 안 해도 됨:**
```
1차: 5단계로 시작
써보니까: "이것도 필요"
2차: 6단계로 수정
→ 점점 정교해짐 (잡돌이 루프)
```

## [6:00-9:00] VALUE 2 - appkit 사례

**중요:**
> **appkit은 정답 아님. 참고 예시임.**

목표: 따라하기 ❌ / 보고 내 방식 만들기 ✅

### appkit이란?

**문제:** "AI 앱 아이디어, 어떻게 구체화?"

**내 사이클:**
```
1. 비슷한 서비스 3-5개 찾기
2. 사용자 그룹 나누기
3. 핵심 기능 5-8개로 쪼개기
4. 각 기능 상세 정리
5. 중복 통합
6. 정책/규칙 명확히
```

→ 이 사이클을 커맨드로 = **appkit**

### 구조

```
/appkit.new     → 아이디어 분해
/appkit.spec    → 기능 구체화
/appkit.merge   → 중복 통합
/appkit.design  → API 설계
/appkit.policy  → 정책 정리
/appkit.verify  → 완성도 체크
```

각 커맨드 = 하나의 작업 사이클

### 실제 예시

**/appkit.new**

```
/appkit.new 테니스장 예약 앱, 쿠폰 할인

→ 자동 분석:
📋 유사 서비스: 야놀자, 카카오맵, 네이버예약
📋 사용자 그룹: 일반/관리자/플랫폼
📋 핵심 기능: user, venue, booking, promotion, review
📋 구현 순서: 의존성 기반

✅ 완료
```

**/appkit.spec**

```
/appkit.spec 003-booking "메인에서 검색"

→ 자동 정리:
1. 사용자 여정
2. 비즈니스 규칙
3. 예외 상황
4. 의존성

✅ spec.md 추가
```

### appkit의 의미

**완성된 시스템 아님**
**사이클 정리 연습의 예시임**

```
제작자 방식을 6개 사이클로 나눔
각 사이클을 커맨드로 만듦
필요할 때 꺼내 씀
```

**여러분이 배울 것:**
```
appkit 정답 ❌
appkit 방식 ✅

→ "사이클은 이렇게 나누는구나"
→ "커맨드는 이렇게 쓰는구나"
→ 내 업무에 적용
```

### 활용 방법

**1. 참고용 (추천)**
```
구조 보고 영감 받기
내 방식으로 만들기
```

**2. 복붙 수정**
```
용어만 바꾸기
"앱" → "영상"
"사용자" → "시청자"
```

**3. 새로 만들기**
```
appkit 보고 내 구조 설계
```
