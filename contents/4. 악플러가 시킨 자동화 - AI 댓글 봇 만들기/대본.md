## [0:30-1:30] 오늘 다룰 범위

먼저 오늘 영상의 범위를 말씀드릴게요.

이 프로젝트는 **Cloudflare Workers**라는 걸로 만들었는데요,
나중에 인터넷에 배포해서 24시간 돌릴 수 있어요.

**근데 오늘은 그거 안 다룹니다.**

오늘은 **내 컴퓨터(localhost)에서 돌리는 것**만 다룰 거예요.

왜냐면:
- 배포까지 다루면 너무 길어짐
- 초심자한테 한 번에 너무 많은 정보
- **일단 돌아가는 거 보는 게 먼저**

Cloudflare가 뭔지, 배포가 뭔지, 서버리스가 뭔지...
**지금 당장 몰라도 됩니다.**

오늘은 그냥 "와, 이런 게 되는구나" 느끼시면 돼요.
배포는 다음 영상에서 차근차근 다룰게요.

## [1:30-3:00] 결과물 먼저 보여주기

자, 완성된 결과물부터 보여드릴게요.

*[화면: localhost:8788 대시보드]*

이게 제가 만든 YouTube 댓글 자동 응답 대시보드입니다.
지금 제 컴퓨터에서 돌아가고 있어요.

**기능 설명:**
1. **댓글 가져오기** - 내 채널의 모든 영상에서 댓글을 수집합니다
2. **AI 분류** - 긍정/부정/질문/제안 등으로 자동 분류
3. **응답 생성** - AI가 댓글마다 맞춤 응답을 만들어줍니다
4. **한 번에 게시** - 버튼 하나로 YouTube에 답글 게시

*[화면: 실제 동작 시연]*

댓글 가져오기 버튼 누르면...
보시는 것처럼 제 채널의 모든 댓글이 쫙 수집됩니다.

여기서 AI가 자동으로 분류해주고요.
응답 생성 버튼 누르면 각 댓글에 맞는 응답이 생성됩니다.

마음에 들면 게시, 아니면 수정할 수 있어요.
**완전 자동은 아니고, 반자동입니다.**
왜냐면 이상한 답글이 올라가면 안 되니까요.

## [3:00-5:00] 초심자를 위한 기술 설명

이제 초심자분들께 이게 어떻게 돌아가는지 설명드릴게요.

### 사용한 것들


**"뭔 소린지 하나도 모르겠다"** 하실 수 있어요.
하나씩 쉽게 설명드릴게요.

---

### 1. Hono - 웹 프레임워크

프레임워크가 뭐냐면요,
**자주 쓰는 기능들을 미리 만들어놓은 도구 모음**이에요.

웹 서비스를 만들 때:
- URL 요청 받기
- 로그인 처리
- 데이터 주고받기

이런 걸 매번 처음부터 만들면 힘드니까,
프레임워크가 "그거 내가 해줄게" 하는 거죠.

```typescript
// 이게 전부예요
app.get('/health', (c) => {
  return c.json({ status: 'ok' })
})
```

"이 주소로 오면 이거 보여줘" - 끝.

---

### 2. KV - 사물함 같은 저장소

**데이터베이스(DB)**는 엑셀 같은 거예요.
행과 열이 있고, 검색하고, 정렬하고, 필터링하고...

**KV(Key-Value)**는 **사물함** 같은 거예요.
- 열쇠(Key)가 있으면 해당 칸(Value)을 열 수 있음
- "123번 사물함 열어줘" → 123번 내용물 나옴
- 끝. 그게 전부.

```
사물함 비유:
- DB: "키 160 이상인 사람 물건 다 꺼내줘" 가능
- KV: "123번 열어줘"만 가능
```

**왜 KV를 썼나요?**
- 댓글 봇은 복잡한 검색이 필요 없음
- 댓글 ID로 조회하면 끝
- 설정이 간단함

DB가 필요해지면 나중에 바꾸면 돼요.
**처음부터 완벽할 필요 없습니다.**

---

### 3. OpenRouter - AI API 중개소

AI를 쓰려면 API를 호출해야 해요.

- OpenAI API - GPT 모델
- Anthropic API - Claude 모델
- Google API - Gemini 모델

근데 각각 가입하고, 카드 등록하고, 키 받고... 귀찮죠?

**OpenRouter는 AI API 중개소**예요.
하나만 가입하면 여러 AI 모델을 쓸 수 있습니다.

이 프로젝트에서는:
- **댓글 분류**: Mistral Nemo (저렴)
- **응답 생성**: Gemini 2.0 Flash (품질)

목적에 따라 모델을 다르게 쓰는 거죠.
분류는 정확도보다 비용이 중요하고,
응답은 비용보다 품질이 중요하니까요.

## [5:00-6:30] localhost로 시작하는 이유

"아니 localhost에서만 돌아가면 뭐해요? 컴퓨터 끄면 끝 아닌가요?"

맞는 말이에요. 근데 **일부러** 이렇게 시작하는 거예요.

### 작게 시작하는 철학

**"작게 시작해서 필요하면 키워라"**

이게 개발의 핵심이에요.

처음부터 "전 세계 유튜버가 쓸 서비스!" 이러면:
- 만들다가 지침
- 아무도 안 씀
- 돈만 날림

대신:
1. **localhost에서 먼저 만듦** ← 오늘 여기
2. 내가 써봄
3. 문제 발견하면 고침
4. 괜찮으면 배포 ← 다음 영상

**이게 MVP(최소 기능 제품) 방식이에요.**

이전 영상에서 말씀드린 것처럼,
Appkit으로 아이디어를 구체화하고,
**MVP 범위를 정한 다음에 만드는 거죠.**

### localhost의 장점

- **빠른 테스트** - 수정하고 바로 확인
- **비용 0원** - 서버비 없음
- **실수해도 OK** - 나만 보니까
- **자유로움** - 마음대로 실험 가능

배포는 localhost에서 충분히 검증한 다음에 해도 늦지 않아요.

## [6:00-8:00] 코드 구조 훑어보기 (초심자용)

코드를 깊게 보진 않을 거예요.
대신 **구조**만 이해해봅시다.

### 폴더 구조

```
src/
├── index.ts          # 메인 (진입점)
├── types.ts          # 타입 정의
├── lib/
│   └── kv.ts         # 데이터 저장/조회
├── routes/
│   └── api.ts        # API 엔드포인트
├── services/
│   ├── youtube.ts    # YouTube API 연동
│   ├── llm.ts        # AI 연동
│   └── scheduled.ts  # 자동 실행
└── views/
    ├── dashboard.ts  # 대시보드 화면
    └── login.ts      # 로그인 화면
```

**비유로 설명하면:**
- `index.ts` = 회사 정문 (모든 요청이 여기로 옴)
- `routes/` = 부서 안내 (어디로 갈지 정함)
- `services/` = 실제 업무 담당 (일하는 곳)
- `views/` = 고객 응대 (화면 보여주는 곳)
- `lib/` = 공용 도구함 (여러 곳에서 쓰는 기능)

### 핵심 흐름

```
1. 사용자가 "댓글 가져오기" 클릭
2. index.ts에서 요청 받음
3. routes/api.ts로 전달
4. services/youtube.ts가 YouTube API 호출
5. lib/kv.ts가 데이터 저장
6. views/dashboard.ts가 결과 화면 표시
```

레고 블록처럼 각자 역할이 있고,
연결해서 하나의 서비스가 되는 거예요.

**공개합니다.**

GitHub에 올려둘게요. 링크는 더보기란에.

### 필요한 것들 (localhost 실행용)

1. **Node.js** - 18버전 이상
2. **YouTube API 키** - Google Cloud Console에서 발급
3. **YouTube OAuth 설정** - 댓글 게시 권한용
4. **OpenRouter API 키** - AI 사용료 (저렴함)

*Cloudflare 계정은 지금 당장은 필요 없어요. 배포할 때 필요합니다.*

### 실행 방법

```bash
# 1. 프로젝트 다운로드
git clone [링크]

# 2. 폴더로 이동
cd youtube-reply-bot

# 3. 의존성 설치
npm install

# 4. 환경변수 설정 (.dev.vars 파일 만들기)
# YOUTUBE_API_KEY=여러분의_키
# YOUTUBE_CHANNEL_ID=여러분의_채널ID
# OPENROUTER_API_KEY=여러분의_키
# ... (자세한 건 README 참고)

# 5. 실행
npm run dev

# 6. 브라우저에서 열기
# http://localhost:8788
```

이렇게 하면 **여러분 컴퓨터에서** 댓글 봇이 돌아갑니다.

**어려우시면 댓글 달아주세요.**
