## [1:30-3:30] VALUE DELIVERY 1 - VS Code가 뭔지 이해하기

### VS Code란 무엇인가?

**VS Code = Visual Studio Code**

- 마이크로소프트가 만든 무료 프로그램
- 텍스트 편집기의 고급 버전
- 개발자들이 가장 많이 쓰는 도구

**쉽게 비유하면:**
- 워드/한글 = 문서 작성 도구
- VS Code = 코드/파일 작성 도구

**왜 이걸 써야 하나?**

1. **파일 관리가 편함** - 프로젝트 폴더 전체를 한눈에 볼 수 있음
2. **Claude와 연동됨** - Claude Code가 VS Code 안에서 작동
3. **무료** - 누구나 쓸 수 있음

### VS Code 실행하기

**실행 방법:**
- Mac: Spotlight 검색 (Cmd+Space) → "VS Code" 입력
- Windows: 시작 메뉴 → "Visual Studio Code"

### VS Code 화면 구성

**화면 구성 (왼쪽에서 오른쪽 순서):**

```
┌─────────────────────────────────────────┐
│ 사이드바 │   편집 영역   │              │
│ (Explorer)│              │              │
│  📁 폴더   │  파일 내용   │              │
│  📄 파일   │  여기서 작성 │              │
├─────────────────────────────────────────┤
│          터미널 (아래 영역)              │
│          나중에 다룰 예정                │
└─────────────────────────────────────────┘
```

**사이드바 (왼쪽):**
- 폴더와 파일 목록
- 프로젝트 구조를 한눈에 볼 수 있음
- "Explorer"라고 부름

**편집 영역 (가운데):**
- 파일 내용 보고 수정하는 곳
- 여러 파일 탭으로 열 수 있음

**터미널 (아래):**
- 명령어 입력하는 곳
- 지금은 몰라도 됨, 나중에 배움

**Claude 패널 (오른쪽):**
- Claude와 대화하는 곳
- 사이드바에서 Claude 아이콘 클릭하면 나옴

### 프로젝트 폴더란?

**프로젝트 = 작업 공간**

"AI 인플루언서 시스템" 만든다면:
- 이 프로젝트 전용 폴더 하나 만들기
- 관련 파일 전부 여기에 모으기
- Claude는 이 폴더 안에서만 작업

**왜 폴더로 관리하나요?**

예시:
```
내 문서/
├─ AI프로젝트1/          ← 프로젝트1 전용
│  ├─ README.md
│  ├─ CLAUDE.md
│  └─ characters/
│
├─ AI프로젝트2/          ← 프로젝트2 전용
│  ├─ README.md
│  └─ content/
```

→ 프로젝트끼리 섞이지 않음
→ Claude가 맥락 정확히 이해
→ 나중에 찾기 쉬움

## [3:30-5:30] VALUE DELIVERY 2 - Claude Code가 뭔지 이해하기

### Claude Code란 무엇인가?

**Claude Code = VS Code에서 작동하는 Claude AI**

**웹사이트 Claude와의 차이:**

| 구분 | 웹사이트 Claude | Claude Code |
|------|----------------|-------------|
| 위치 | 브라우저 | VS Code 안 |
| 파일 접근 | 복사-붙여넣기 필요 | 자동으로 읽음 |
| 작업 방식 | 대화만 | 파일 직접 수정 가능 |
| 맥락 유지 | 대화 기록만 | 프로젝트 전체 파일 |

**왜 Claude Code를 써야 하나?**

**예시로 이해하기:**

웹 Claude:
```
나: (블로그 글 복사-붙여넣기) 이거 수정해줘
Claude: (수정본 생성)
나: (다시 복사-붙여넣기)
```

Claude Code:
```
나: @blog.md 이 파일 수정해줘
Claude: (파일 직접 읽고 수정함)
```

**핵심: 파일 기반 작업이 편함**

### Claude Code 시작하기

**실행 방법:**

1. VS Code에서 프로젝트 폴더 열기
2. 터미널 열기 (Ctrl+` 또는 Cmd+`)
3. 입력:

```bash
claude
```

또는 빠른 모드:

```bash
claude --dangerously-skip-permissions
```

**권장: 빠른 모드 사용**
- 매번 권한 확인 안 해도 됨
- 본인 프로젝트니까 안전함

**Claude 패널이 오른쪽에 나타남**

### 웹 Claude vs Claude Code 언제 뭘 쓰나?

**웹 Claude를 쓸 때:**
- 간단한 질문 ("이게 뭐야?")
- 아이디어 브레인스토밍
- 파일 작업 없는 대화

**Claude Code를 쓸 때:**
- 파일 여러 개 다루는 작업
- 프로젝트 단위 작업
- 지침을 계속 쌓아가는 작업 (잡도리 루프!)

**→ 업무 매뉴얼 만들기는 무조건 Claude Code**

### Step 4: md 파일이 뭔가요?

**md = Markdown 파일**

**Markdown이란?**
- 간단한 문서 작성 형식
- 메모장처럼 그냥 한글로 편하게 쓰면 됨
- GitHub, 노션 등에서 널리 사용

**문법 몰라도 됩니다.**

그냥 한글로 편하게 쓰세요.
Claude에게 부탁하면예쁘게 정리해줍니다.

**왜 md 파일을 쓰나요?**

1. **Claude가 잘 읽음**: 구조화된 형식이라 맥락 이해 쉬움. ppt나 이미지 같은 파일은 AI가 이해하기가 어렵습니다. AI의 업무 효율을 위해 md를 씁니다.
2. **간단함**: 복잡한 워드 문서보다 가볍고 빠름 

**중요:**
Claude는 md 파일을 읽어서 프로젝트 맥락을 이해합니다.

## [5:30-7:30] VALUE DELIVERY 3 - CLAUDE.md가 핵심인 이유

### CLAUDE.md = AI 직원의 업무 매뉴얼

**이 파일이 전부입니다.**

Claude가 이 프로젝트를 도울 때:
1. CLAUDE.md를 읽음
2. 규칙을 이해함
3. 그대로 따라함

**이게 바로 1인 AI 조직의 핵심입니다.**

직원(AI)한테 명확한 매뉴얼을 주는 것.

### CLAUDE.md에 뭘 쓰나요?

**CLAUDE.md = 목차 (인덱스)**

너무 많은 규칙을 한 파일에 쓰지 마세요.
규칙이 길어지면 찾기 어렵고 관리 힘듦.

**추천 구조:**

```
my-project/
├─ CLAUDE.md          ← 마스터 인덱스 (목차만)
└─ rules/             ← 세부 규칙들
   ├─ character.md    ← 캐릭터 생성 규칙
   ├─ content.md      ← 콘텐츠 작성 규칙
   └─ tone.md         ← 톤앤매너 규칙
```

**CLAUDE.md (간단하게):**
```markdown
# 프로젝트 설명

AI 인플루언서 100명을 운영하는 시스템
- 각 인플루언서는 고유한 캐릭터
- 플랫폼별 콘텐츠 자동 생성

## 작업 규칙

세부 규칙은 rules/ 폴더 참고:
- @rules/character.md - 캐릭터 생성 규칙
- @rules/content.md - 콘텐츠 작성 규칙
- @rules/tone.md - 톤앤매너 규칙
```

**rules/character.md (상세하게):**
```markdown
# 캐릭터 생성 규칙

## 나이 분포
- 10대: 20%
- 20대: 50%
- 30대: 30%

## 직업 다양성
- 프로페셔널: 40%
- 크리에이터: 30%
- 학생/취미: 30%

## 금지 사항
- 이름 중복 금지
- 같은 직업 3명 이상 금지
```

**왜 이렇게 나누나요?**

- CLAUDE.md는 짧고 간단하게 (목차 역할)
- 세부 규칙은 rules/ 폴더에 분리
- 필요할 때 `@rules/character.md` 이렇게 참조
- 규칙 찾기 쉽고 관리 편함

3. 일 시키기
→ 체계적으로 일 잘됨

**CLAUDE.md = AI 직원의 업무 매뉴얼**

이게 전부입니다.

**핵심:**

CLAUDE.md에 작업 방식을 명확히 적어두면,

### @ 태그 - Claude에게 파일 보여주기

**핵심 기능: @ 태그**

Claude한테 **"이 파일을 읽고 답해줘"**라고 지시하는 방법.

**사용법:**

```
@CLAUDE.md 이 프로젝트를 진행하려면 다음 단계는?
```

**@ 입력하면:**
- 파일 목록이 자동으로 나타남
- 화살표 키로 선택하거나
- 파일명 타이핑해서 검색

**@ 태그가 편한 이유:**

```
@CLAUDE.md 다음 단계는?
@characters/ 폴더 내용 확인해줘
```

→ 파일 내용을 직접 Claude에게 전달
→ 맥락을 정확히 이해하고 답변

**중요: CLAUDE.md는 Claude 시작 시 자동으로 읽힙니다.**

그래도 @CLAUDE.md를 명시적으로 쓰면:
- Claude가 다시 한번 확인
- 최신 내용 반영
- 더 정확한 답변

### 실습: 첫 질문 해보기
