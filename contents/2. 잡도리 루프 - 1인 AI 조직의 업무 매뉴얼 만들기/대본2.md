## [1:30-3:00] VALUE DELIVERY 1 - VS Code가 뭔지 이해하기

### VS Code란 무엇인가?

**VS Code = Visual Studio Code**

- 마이크로소프트가 만든 무료 프로그램
- 텍스트 편집기의 고급 버전
- 개발자들이 가장 많이 쓰는 도구

**쉽게 비유하면:**
- 워드/한글 = 문서 작성 도구
- VS Code = 코드/파일 작성 도구

**왜 이걸 써야 하나?**

1. **파일 관리가 편함** - 프로젝트 폴더 전체를 한눈에 볼 수 있음
2. **Claude와 연동됨** - Claude Code가 VS Code 안에서 작동
3. **무료** - 누구나 쓸 수 있음

### VS Code 실행하기

**실행 방법:**
- Mac: Spotlight 검색 (Cmd+Space) → "VS Code" 입력
- Windows: 시작 메뉴 → "Visual Studio Code", 아니면 특정 폴더에서 마우스 우클릭 → "Visual Studio Code"

### VS Code 화면 구성

**화면 구성 (왼쪽에서 오른쪽 순서):**

```
┌─────────────────────────────────────────┐
│ 사이드바 │   편집 영역   │              │
│ (Explorer)│              │              │
│  📁 폴더   │  파일 내용   │              │
│  📄 파일   │  여기서 작성 │              │
├─────────────────────────────────────────┤
│          터미널 (아래 영역)              │
│          나중에 다룰 예정                │
└─────────────────────────────────────────┘
```

**사이드바 (왼쪽):**
- 폴더와 파일 목록
- 프로젝트 구조를 한눈에 볼 수 있음
- "Explorer"라고 부름

**편집 영역 (가운데):**
- 파일 내용 보고 수정하는 곳
- 여러 파일 탭으로 열 수 있음

**터미널 (아래):**
- 명령어 입력하는 곳
- Claude Code를 여기서 실행합니다

### 프로젝트 폴더란?

**프로젝트 = 작업 공간**

"AI 인플루언서 시스템" 만든다면:
- 이 프로젝트 전용 폴더 하나 만들기
- 관련 파일 전부 여기에 모으기
- Claude는 이 폴더 안에서만 작업

**왜 폴더로 관리하나요?**

예시:
```
내 문서/
├─ AI프로젝트1/          ← 프로젝트1 전용
│  ├─ README.md
│  ├─ CLAUDE.md
│  └─ characters/
│
├─ AI프로젝트2/          ← 프로젝트2 전용
│  ├─ README.md
│  └─ content/
```

→ 프로젝트끼리 섞이지 않음
→ Claude가 맥락 정확히 이해
→ 나중에 찾기 쉬움

## [3:00-4:00] VALUE DELIVERY 2 - Claude Code가 뭔지 이해하기

### Claude Code란 무엇인가?

**Claude Code = 터미널 환경에서 작동하는 Claude AI**

**웹사이트 Claude와의 차이:**

| 구분 | 웹사이트 Claude | Claude Code |
|------|----------------|-------------|
| 위치 | 브라우저 | 터미널 안 |
| 파일 접근 | 복사-붙여넣기 필요 | 자동으로 읽음 |
| 작업 방식 | 대화만 | 파일 직접 수정 가능 |
| 맥락 유지 | 대화 기록만 | 프로젝트 전체 파일 |

**왜 Claude Code를 써야 하나?**

**예시로 이해하기:**

웹 Claude:
```
나: (블로그 글 복사-붙여넣기) 이거 수정해줘
Claude: (수정본 생성)
나: (다시 복사-붙여넣기)
```

Claude Code:
```
나: @blog.md 이 파일 수정해줘
Claude: (파일 직접 읽고 수정함)
```

**핵심: 파일 기반 작업이 편함**

### Step 3: Claude와 대화하며 CLAUDE.md 만들기

이제 Claude에게 프로젝트 아이디어를 말해봅시다:

```
나는 AI로 인스타그램 콘텐츠를 자동으로 만들고 싶어.
매일 아침 9시에 자동으로 포스팅되는 시스템을 만들려고 해.
```

**Claude가 질문합니다:**
- 어떤 주제의 콘텐츠인지
- 타겟 오디언스는 누구인지
- 어떤 스타일로 만들 것인지

대화를 좀 나누다가, 이렇게 요청해봅시다:

```
좋아. 그럼 우리가 지금 나눈 대화를 바탕으로
이 프로젝트의 CLAUDE.md 파일을 한글로 만들어줘.
```

**Claude가 CLAUDE.md 파일을 자동으로 만들어줍니다!**

왼쪽 사이드바에 `CLAUDE.md` 파일이 생기고,
방금 나눈 대화 내용이 정리되어 있습니다.

**중요: Claude가 시작할 때 CLAUDE.md를 자동으로 읽습니다.**
영어로 만들면 토큰이 절감되서 비용을 절감할 수 있는데, 지금은 처음이니 문서는 한글로 만듭시다.
클로드 요금제는 많이 쓴다고 비용이 더 나오는게 아니라, 사용제한이 있기때문에 돈이 많이 과금될까봐 걱정할 필요는 없습니다.

- Claude 실행 → CLAUDE.md 찾음
- 있으면 → 자동으로 읽고 규칙 이해
- 없으면 → 일반 모드로 시작

**→ CLAUDE.md = Claude의 작업 지침서**

### Step 4: md 파일이 뭔가요?

**md = Markdown 파일**

**Markdown이란?**
- 간단한 문서 작성 형식
- 메모장처럼 그냥 한글로 편하게 쓰면 됨
- GitHub, 노션 등에서 널리 사용

**문법 몰라도 됩니다.**

그냥 한글로 편하게 쓰세요.
Claude에게 부탁하면 예쁘게 정리해줍니다.

**왜 md 파일을 쓰나요?**

1. **Claude가 잘 읽음**: 구조화된 형식이라 맥락 이해 쉬움. ppt나 이미지 같은 파일은 AI가 이해하기 어렵습니다. AI의 업무 효율을 위해 md를 씁니다.
2. **간단함**: 복잡한 워드 문서보다 가볍고 빠름

**중요:**
Claude는 md 파일을 읽어서 프로젝트 맥락을 이해합니다.

## [6:00-7:00] VALUE DELIVERY 4 - CLAUDE.md가 핵심인 이유

### CLAUDE.md = AI 직원의 업무 매뉴얼

**이 파일이 전부입니다.**

Claude가 이 프로젝트를 도울 때:
1. CLAUDE.md를 읽음
2. 규칙을 이해함
3. 그대로 따라함

**이게 바로 1인 AI 조직의 핵심입니다.**

직원(AI)한테 명확한 매뉴얼을 주는 것.

### CLAUDE.md에 뭘 쓰나요?

**처음에는 규칙만 쓰면 됩니다.**

CLAUDE.md는 나중에 commands, skills, subAgent 같은 고급 기능도 담을 수 있지만,
**지금은 일단 "규칙"만 쓰는 것부터 시작합시다.**

**CLAUDE.md = 규칙들의 목차 (인덱스)**

규칙이 많아지면 한 파일에 다 쓰지 말고,
`rules/` 폴더를 만들어서 규칙별로 파일을 나누세요.

**중요: CLAUDE.md를 수정하면?**

CLAUDE.md는 Claude가 시작할 때 딱 한 번 읽습니다.

따라서:
- CLAUDE.md 수정 → Claude 재시작 필요
- 터미널에서 `Ctrl+C` (종료)
- 다시 `claude --dangerously-skip-permissions` (실행)

**규칙 파일(rules/*)은 다릅니다:**
- rules/character.md 같은 파일은 @태그로 참조할 때마다 새로 읽음
- 재시작 필요 없음

**추천 구조:**

```
my-project/
├─ CLAUDE.md          ← 프로젝트 소개 + 규칙 목차
└─ rules/             ← 세부 규칙들
   ├─ character.md    ← 캐릭터 생성 규칙
   ├─ content.md      ← 콘텐츠 작성 규칙
   └─ tone.md         ← 톤앤매너 규칙
```

**CLAUDE.md 예시 (간단하게):**
```markdown
# AI 인플루언서 시스템

## 프로젝트 개요
AI 인플루언서 100명을 운영하는 시스템
- 각 인플루언서는 고유한 캐릭터
- 플랫폼별 콘텐츠 자동 생성

## 작업 규칙

세부 규칙은 rules/ 폴더 참고:
- rules/character.md - 캐릭터 생성 규칙
- rules/content.md - 콘텐츠 작성 규칙
- rules/tone.md - 톤앤매너 규칙
```

**rules/character.md 예시 (상세하게):**
```markdown
# 캐릭터 생성 규칙

## 나이 분포
- 10대: 20%
- 20대: 50%
- 30대: 30%

## 직업 다양성
- 프로페셔널: 40%
- 크리에이터: 30%
- 학생/취미: 30%

## 금지 사항
- 이름 중복 금지
- 같은 직업 3명 이상 금지
```

**왜 이렇게 나누나요?**

- CLAUDE.md는 짧고 간단하게 (목차 역할)
- 세부 규칙은 rules/ 폴더에 분리
- 필요할 때 `@rules/character.md` 이렇게 참조
- 규칙 찾기 쉽고 관리 편함

## [7:00-8:30] VALUE DELIVERY 5 - @ 태그와 파일 기반 작업

### @ 태그 - Claude에게 파일 보여주기

**핵심 기능: @ 태그**

Claude한테 **"이 파일을 읽고 답해줘"**라고 지시하는 방법.

**사용법:**

```
@CLAUDE.md 이 프로젝트를 진행하려면 다음 단계는?
```

**@ 입력하면:**
- 파일 목록이 자동으로 나타남
- 화살표 키로 선택하거나
- 파일명 타이핑해서 검색

**@ 태그가 편한 이유:**

```
@CLAUDE.md 다음 단계는?
@characters/ 폴더 내용 확인해줘
```

→ 파일 내용을 직접 Claude에게 전달
→ 맥락을 정확히 이해하고 답변

### @ 태그 활용 팁

**팁 1: 특정 규칙 파일 참조**

```
@rules/character.md 캐릭터 3개 만들어줘
```
→ 캐릭터 생성 규칙만 참조해서 작업

**팁 2: 여러 규칙 동시에 참조**

```
@rules/character.md @rules/tone.md
이 두 규칙 모두 적용해서 캐릭터 만들어줘
```

**팁 3: 폴더 전체 참조**

```
@characters/ 이 폴더의 캐릭터들 분석해줘
@rules/ 모든 규칙 확인하고 문제점 찾아줘
```

**팁 4: CLAUDE.md는 보통 생략 가능**

```
캐릭터 3개 만들어줘
```
→ Claude가 CLAUDE.md를 이미 읽었으니 rules/ 폴더를 알고 있음

**팁 5: 파일 없이 그냥 질문도 가능**

```
md 파일 문법 알려줘
```
→ 파일 참조 없이도 일반 질문 가능

**팁 6: 규칙 추가할 때**

```
@rules/character.md
여기에 "이모지 사용 금지" 규칙 추가해줘
```
→ 특정 규칙 파일만 수정

### 파일로 맥락 유지하기

**핵심 개념:**

Claude는 대화만으로는 맥락을 완벽히 기억 못함.

**파일에 중요한 정보 저장 → Claude가 언제든 다시 읽음**

**예시:**

대화로만:
```
나: 캐릭터 나이는 10-30대로 해줘
Claude: (답변)
...
(10분 뒤)
나: 캐릭터 만들어줘
Claude: (나이 설정 까먹음)
```

파일 기반:
