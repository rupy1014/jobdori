# SEO Blog Writer (SEO 블로그 작성기)

유튜브 나레이션 대본을 검색 엔진 최적화(SEO)된 블로그 게시글로 자동 변환합니다.

## 📁 Output

**Output File Naming**: `[원본파일명]_블로그.md`
- 예: `AI 수익화 아이디어.md` → `AI 수익화 아이디어_블로그.md`

**Output Location**: 원본 파일과 동일한 디렉토리

---

## 🎯 Core Purpose

**유튜브 대본 → SEO 블로그 글 변환**
- 구어체 → 문어체 전환 (자연스러운 읽기 경험)
- SEO 키워드 최적화 (검색 상위 노출)
- 블로그 친화적 구조 (H1-H3 계층, 단락 분리)
- 독자 몰입 유지 (스캔 가능한 레이아웃)

**NOT for**:
- 단순 복사-붙여넣기 (X)
- 키워드만 채운 저품질 글 (X)
- 유튜브 스크립트 그대로 옮기기 (X)

## 🔄 변환 프로세스

### Step 1: 대본 분석
```
원본 대본 읽기 →
1. 파일명 추출
2. 핵심 주제 파악
3. 구조 분석 (HOOK, PROBLEM, SOLUTION 등)
4. 타겟 키워드 추출 (메인 3개, 서브 5-7개)
5. 대상 독자 파악
```

### Step 2: SEO 키워드 전략
```
메인 키워드 선정 (3개):
- 검색량 높은 핵심 주제어
- 예: "AI 수익화", "Claude Code", "비개발자"

서브 키워드 (5-7개):
- 롱테일 키워드 (구체적 검색어)
- 예: "AI로 돈 버는 법", "집에서 AI 부업"

LSI 키워드 (의미적 연관어):
- 검색 엔진이 주제 이해하도록
- 예: "생성형 AI", "프롬프트 엔지니어링", "자동화"
```

### Step 3: 블로그 구조 재구성
```
유튜브 대본 구조 → 블로그 구조

[대본]                    [블로그]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
HOOK (0-15s)         → 도입 (H1 제목 + 리드 문단)
PROBLEM (15-45s)     → 문제 제기 (H2)
AGITATE (45s-1:30)   → 문제 심화 (H2)
SOLUTION (1:30-2:00) → 솔루션 소개 (H2)
VALUE (2:00-8:00)    → 본문 (H2, H3 계층 구조)
PROOF (8:00-9:00)    → 사례/증거 (H2)
CTA (9:00-10:00)     → 결론 + CTA (H2)
```

### Step 4: 문체 변환
```
구어체 → 문어체 자연스럽게 변환

[Before - 대본 구어체]
"여러분, 혹시 이런 경험 있으신가요?
AI 써봤는데 별로더라...
그래서 포기하신 분들 많으시죠?"

[After - 블로그 문어체]
많은 사람들이 AI를 처음 접하고 실망합니다.
기대만큼 결과가 나오지 않았기 때문입니다.
하지만 포기하기엔 너무 이릅니다.
```

### Step 5: SEO 메타데이터 생성
```
블로그 상단에 포함:
1. SEO 제목 (Title Tag) - 60자 이내
2. 메타 설명 (Meta Description) - 160자 이내
3. 메인 키워드 리스트
4. 카테고리/태그
5. 예상 읽기 시간
```

### Step 6: 블로그 최적화 요소 추가
```
1. 목차 (TOC) - 긴 글 (2000자+)인 경우
2. 이미지 삽입 지점 - [이미지: 설명] 형태로 제안
3. 인용구 (Blockquote) - 핵심 메시지 강조
4. 리스트 (Bullet/Numbered) - 스캔 가능성 향상
5. 내부 링크 제안 - 관련 글 연결 (있는 경우)
6. CTA 버튼 - 구독, 다운로드 등
```

### Step 7: 파일 생성
```
1. 출력 파일명: [원본파일명]_블로그.md
2. Write 도구로 md 파일 생성
3. 절대 경로 사용
4. UTF-8 인코딩
5. 생성 완료 안내
```

## 🎯 SEO 최적화 체크리스트

### 제목 최적화
- [ ] H1 태그 1개만 사용 (제목)
- [ ] 메인 키워드 제목 앞쪽 배치
- [ ] 60자 이내
- [ ] 감정적 트리거 포함 (방법, 완벽, 가이드 등)

### 본문 최적화
- [ ] 메인 키워드 밀도 1-2%
- [ ] 첫 100단어 내 메인 키워드 등장
- [ ] H2 제목마다 서브 키워드 포함
- [ ] 단락 길이 3-4줄 (모바일 가독성)
- [ ] 리스트/표 활용 (스캔 가능성)

### 메타데이터 최적화
- [ ] Meta Description 160자 이내
- [ ] 메인 키워드 포함
- [ ] 명확한 가치 제안
- [ ] CTA 포함 (클릭 유도)

### 이미지 최적화
- [ ] 대표 이미지 삽입 지점 제안
- [ ] 본문 2-3개 이미지 제안
- [ ] Alt Text 제안 (키워드 포함)
- [ ] 이미지 캡션 제안

### 링크 최적화
- [ ] 내부 링크 2-3개 제안
- [ ] 외부 링크 1-2개 (권위 있는 소스)
- [ ] Anchor Text 키워드 최적화

### 가독성 최적화
- [ ] 목차 제공 (2000자 이상)
- [ ] 소제목 명확 (H2, H3)
- [ ] 짧은 문장 (20단어 이하)
- [ ] 능동태 사용
- [ ] 전문 용어 최소화

## ✍️ 문체 변환 가이드

### 구어체 → 문어체 변환 규칙

#### 규칙 1: 반말 제거, 존댓말 유지
```
[대본]
"여러분~ 오늘은요, 진짜 중요한 얘기 할게요!"

[블로그]
오늘은 중요한 내용을 다루겠습니다.
```

#### 규칙 2: 불필요한 추임새 제거
```
[대본]
"그러니까, 뭐냐면, 사실은요..."

[블로그]
핵심은 다음과 같습니다.
```

#### 규칙 3: 반복 제거, 한 번만 명확하게
```
[대본]
"AI 학습시켜야 해요. 학습시켜야 합니다.
맥락을 학습시켜야 해요."

[블로그]
AI에게 맥락을 학습시켜야 합니다.
```

#### 규칙 4: 질문형 → 서술형
```
[대본]
"왜 그럴까요? 바로 이거예요!"

[블로그]
그 이유는 다음과 같습니다.
```

#### 규칙 5: 감정 표현 → 객관적 서술
```
[대본]
"진짜 대박이에요! 완전 놀라울 거예요!"

[블로그]
효과적인 방법입니다. 주목할 만한 결과를 얻을 수 있습니다.
```

## 🔍 키워드 추출 전략

### 메인 키워드 선정 (3개)

**기준:**
1. 검색량 (네이버/구글 키워드 플래너)
2. 경쟁도 (낮을수록 좋음)
3. 대본의 핵심 주제

**예시:**
대본 주제: "AI로 수익 내는 법"
- 메인 키워드 1: AI 수익화
- 메인 키워드 2: AI 부업
- 메인 키워드 3: 생성형 AI 활용

### 서브 키워드 (5-7개)

**롱테일 키워드:**
- "AI로 돈 버는 법"
- "집에서 AI 부업하기"
- "비개발자 AI 활용"
- "ChatGPT 수익화"
- "Claude Code 사용법"

### LSI 키워드 (의미 연관어)

**검색 엔진이 주제 이해하도록:**
- 프롬프트 엔지니어링
- AI 자동화
- 생산성 향상
- 워크플로우 최적화
- 콘텐츠 생성

## 🖼️ 이미지 삽입 가이드

### 필수 이미지 위치

**1. 대표 이미지 (상단)**
- 위치: H1 제목 바로 아래
- 용도: 썸네일, SNS 공유 이미지
- 제안 형식: `[이미지: 설명 + Alt Text]`

**2. 본문 이미지 (2-3개)**
- H2 섹션마다 1개
- 복잡한 개념 설명 시
- 프로세스/비교 시각화

**3. CTA 이미지 (하단)**
- 다운로드/구독 유도
- 버튼 형태 또는 배너

### 이미지 제안 포맷
```markdown
[이미지: AI 자동화 프로세스 플로우차트]
Alt Text: "AI 워크플로우 4단계 - 태스크 분석부터 자동화까지"
캡션: AI 자동화는 4단계로 진행됩니다.
```

2. 키워드를 문맥에 맞게 배치

## 🔧 실행 가이드 (Agent Implementation)

### 파일 생성 프로세스 (Pseudo-code)

```python
# 1. 대본 파일 읽기
source_file = "AI 수익화 아이디어.md"
source_content = Read(source_file)

# 2. 대본 분석
analysis = analyze_script(source_content)
# returns: {
#   topic: "AI 수익화",
#   target_audience: "비개발자",
#   main_keywords: ["AI 수익화", "AI 부업", "Claude Code"],
#   sub_keywords: ["AI로 돈 버는 법", "ChatGPT 활용", ...],
#   sections: [{type: "HOOK", content: "..."}, ...],
#   word_count: 2500
# }

# 3. SEO 키워드 전략
keywords = extract_keywords(analysis)
# Main (3): primary search terms
# Sub (5-7): long-tail keywords
# LSI: semantic related terms

# 4. 블로그 구조 재구성
blog_structure = restructure_for_blog(analysis.sections)
# HOOK → H1 + Lead paragraph
# PROBLEM → H2
# VALUE → H2 + H3 subsections
# PROOF → H2
# CTA → H2 + CTA buttons

# 5. 문어체 변환
blog_content = convert_to_written_style(blog_structure)
# 구어체 → 문어체
# 반말 제거, 추임새 제거
# 질문형 → 서술형

# 6. SEO 메타데이터 생성
metadata = generate_seo_metadata(keywords, analysis)
# title (60자 이내)
# description (160자 이내)
# keywords, category, tags

# 7. 블로그 최적화 요소 추가
optimized = add_blog_elements(blog_content)
# 목차 (TOC)
# 이미지 삽입 지점 [이미지: 설명]
# 인용구 (핵심 메시지)
# 리스트 (Bullet/Numbered)
# 내부 링크 제안

# 8. 최종 출력 파일 생성
output_filename = source_file.replace(".md", "_블로그.md")
output_content = f"""---
title: "{metadata.title}"
description: "{metadata.description}"
keywords: {metadata.keywords}
category: "{metadata.category}"
tags: {metadata.tags}
reading_time: "{metadata.reading_time}"
published_date: {today}
---

# {metadata.h1_title}

{optimized.lead_paragraph}

{optimized.main_image}

---

## 목차

{optimized.toc}

---

{optimized.body}

---

## 결론

{optimized.conclusion}

**다음 단계:**
{optimized.cta}

---

**관련 글:**
{optimized.internal_links}

**태그:** {metadata.hashtags}
"""

# 9. 파일 쓰기
Write(output_filename, output_content)

# 10. 통계 생성 및 안내
stats = calculate_blog_stats(output_content)
print(f"""
✨ 완료!

생성된 파일: {output_filename}

📊 블로그 통계:
- 글자 수: {stats.word_count}자
- 예상 읽기 시간: {stats.reading_time}분
- 키워드 밀도: {stats.keyword_density}% (적정)
- 이미지 제안: {stats.image_count}개
- 내부 링크 제안: {stats.internal_link_count}개
""")
```

### 중요 구현 노트

1. **파일명 생성**: 원본 파일명에서 `.md` 제거 후 `_블로그.md` 추가
2. **경로 유지**: 원본 파일과 동일한 디렉토리에 생성
3. **Write 도구 사용**: Claude의 Write 도구로 파일 생성
4. **절대 경로**: Write 도구에 절대 경로 전달 필요
5. **인코딩**: UTF-8 인코딩 사용 (한글 지원)
6. **키워드 밀도**: 1-2% 유지 (과도하면 페널티)
7. **가독성**: Flesch Reading Ease 60+ 목표

### 에러 처리

- 원본 파일 없음: "파일을 찾을 수 없습니다" 안내
- 파일 쓰기 실패: 권한 확인 또는 경로 확인 안내
- 대본 구조 이상: "대본 형식을 인식할 수 없습니다" 안내
- 키워드 추출 실패: "주제를 파악할 수 없습니다" 안내
