>
> 문제는 **2-3단계가 혼자서는 정말 어렵다**는 겁니다.
>
> 내 머릿속 모호한 느낌을 명확한 기능으로 바꾸려면?
> **대화 상대가 필요합니다.**

## [5-10분] WHY - 구체화가 어려운 진짜 이유

### 슬라이드 4: 왜 아이디어 구체화가 어려울까?

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   구체화가 어려운 3가지 이유
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1️⃣ 혼자서는 객관화가 안 됨
   내 머릿속: "완벽한 아이디어! 😍"
   현실: "이게 뭔지 모르겠는데?" 🤔

   → 피드백이 필요한데 물어볼 사람이 없음

2️⃣ 뭘 모르는지 모름
   "리서치를 해야 한다" → 뭘 리서치하지?
   "경쟁사 분석 필요" → 경쟁사가 뭐지?
   "사용자 정의 필요" → 누가 쓸까?

   → 모르는 걸 모르는 상태

3️⃣ 시간이 지날수록 흐려짐
   Day 1: "대박 아이디어!" 🔥
   Day 7: "이게 뭐였지?" 😅
   Day 30: "그냥 포기..." 😔

   → 열정은 식고 아이디어는 흐려짐

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

핵심: 지속적인 대화 파트너가 필요
```

**스크립트**:
> 구체화가 어려운 이유 세 가지입니다.
>
> 첫째, **혼자서는 객관화가 안 됩니다.**
> 내 머릿속엔 완벽한데, 남들은 이해 못 하죠.
>
> 둘째, **뭘 모르는지 모릅니다.**
> 리서치하래서 구글 켰는데... 뭘 검색해야 하나?
>
> 셋째, **시간이 지날수록 흐려집니다.**
> 처음엔 대박 아이디어였는데, 일주일 지나면 기억도 안 나죠.
>
> 결국 **지속적으로 대화하며 구체화를 도와줄 파트너**가 필요합니다.

>

> 컨설팅은 너무 비싸고,
> 혼자 하면 방향을 못 잡죠.
>
> AI를 활용하면?
>
> **24시간 대화 가능**, 새벽 3시에 아이디어 떠올라도 바로 구체화 시작.
>
> **즉각적인 피드백**, "이거랑 비슷한 서비스 뭐 있어?" 바로 답변.
>
> **모든 대화가 문서로 저장**, 한 달 후에도 이어서 진행 가능.
>
> 월 2만원이면 최고의 구체화 파트너를 얻는 겁니다.

## [10-18분] HOW - AI와 대화하며 구체화하기

### 슬라이드 6: 실제 구체화 데모

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   실제 데모: 모호한 아이디어 구체화
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

시작: 막연한 불편함
"혼자 밥 먹기 싫어" 😔

목표: 구체적인 서비스
- 핵심 기능 정의
- 사용자 그룹 명확화
- 수익 모델 설정
- 개발 가능한 명세

과정: AI와 대화
- 왜 불편한지 탐색
- 비슷한 서비스 조사
- 차별점 찾기
- 기능으로 구체화
```

**스크립트**:
> 실제로 해보겠습니다.
>
> "혼자 밥 먹기 싫어"라는 막연한 불편함에서 시작해보죠.
>
> 이걸 어떻게 구체적인 서비스로 만들까요?

---

### 슬라이드 7: 구체화 대화 - 1단계 (탐색)

**터미널 화면**:
```bash
$ claude

User: 혼자 밥 먹는 게 싫어서 앱을 만들고 싶어.
     근데 뭘 만들어야 할지 모르겠어.

Claude: 혼자 밥 먹는 게 싫은 이유를 좀 더 구체적으로
       알려주실 수 있나요?

       - 대화 상대가 없어서?
       - 식당 가기 어색해서?
       - 메뉴 정하기 어려워서?
       - 다른 이유?

User: 음... 혼밥하면 대화도 없고,
     맛집 가기도 어색하고, 그냥 외로워.

Claude: 이해합니다. 그럼 이런 방향은 어떨까요?

       1. 밥친구 매칭 앱
          - 같은 시간, 같은 지역 매칭
          - 관심사 기반 매칭

       2. 온라인 밥친구
          - 화상으로 함께 식사
          - 먹방 스트리밍 공유

       3. 혼밥러 커뮤니티
          - 혼밥 맛집 공유
          - 혼밥 꿀팁 교환

       어떤 방향이 끌리시나요?

User: 1번! 실제로 만나서 같이 먹는 거.
```

**스크립트**:
> AI가 제 모호한 불편함을 구체적인 방향으로 이끌어줍니다.
>
> "혼밥 싫어" → "왜 싫어?" → "외로워" → "그럼 매칭?"
>
> 자연스럽게 서비스 방향이 잡히죠.

---

### 슬라이드 8: 구체화 대화 - 2단계 (리서치)

**터미널 화면**:
```bash
User: 밥친구 매칭 앱으로 가자.
     비슷한 서비스 있나?

Claude: 네, 몇 가지 유사 서비스가 있습니다:

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📱 기존 서비스 분석
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 소셜다이닝 (해외)
   - Eatwith, VizEat
   - 홈파티 중심, 여행자 타겟

2. 모임 앱 활용 (국내)
   - 소모임, 프립
   - 밥모임도 있지만 주력 아님

3. 데이팅 앱 변형
   - 번개장터 "동네친구"
   - 밥 약속은 부가 기능

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

차별화 포인트 제안:
✅ 밥 약속 전용 (집중)
✅ 동네 중심 (접근성)
✅ 부담 없는 만남 (1시간 점심)
✅ 안전 장치 (공개 장소만)

User: 오 좋아! 그럼 핵심 기능 뭐가 필요할까?
```

**스크립트**:
> AI가 자동으로 시장 조사를 해줍니다.
>
> 비슷한 서비스, 차별화 포인트까지 제안하네요.
>
> 혼자였다면 이런 리서치에 며칠 걸렸을 겁니다.

## Q&A 대비

### 예상 질문과 답변

**Q1: AI가 만든 제품은 차별화가 어렵지 않나요?**
> AI는 도구일 뿐입니다. 포토샵이 있다고 모두가 디자이너가 되는 건 아니죠.
> 차별화는 여러분의 독특한 관점과 경험에서 나옵니다.
> AI는 그걸 빠르게 구현하도록 도와줄 뿐입니다.

**Q2: 코딩을 전혀 모르는데도 가능한가요?**
> 네, 가능합니다. 바이브코딩은 자연어로 지시하면 코드를 생성합니다.
> "버튼을 빨간색으로 바꿔줘" 이렇게 말하면 됩니다.
> 다만 기본적인 웹/앱 동작 원리는 이해하면 더 좋습니다.

**Q3: 구체화했는데 이미 있는 서비스면 어떻게 하나요?**
> 오히려 좋은 신호입니다. 시장이 검증됐다는 뜻이죠.
> 차별화 포인트를 찾아 니치 마켓을 공략하세요.
> 배민이 있어도 요기요가 성공한 것처럼요.

**Q4: 실패하면 어떻게 하나요?**
> 2주 투자니까 실패해도 괜찮습니다.
> 오히려 빠른 실패가 빠른 학습입니다.
> 3개 만들면 1개는 성공한다고 봅니다.

**Q5: 투자나 팀빌딩은 어떻게 하나요?**
> 먼저 혼자서 MVP를 만들고 트래픽을 만드세요.
> 실제 사용자가 있으면 투자도, 팀원도 자연스럽게 따라옵니다.
> 아이디어만으로는 아무도 설득할 수 없습니다.
